## СЛАЙД 2: Проблема (показать плохой пример)

```
❌ ПЛОХОЙ ПРОМПТ:

"Напиши статью про маркетинг"
```

**Демонстрация:**
1. Откройте ChatGPT
2. Введите этот промпт
3. Покажите результат: общий, размытый, не то что нужно

**Что говорить:**
"Смотрите, что происходит. Модель выдала что-то, но это не совсем то, что мы хотели. Почему? Потому что промпт неструктурированный. Модель угадывает ваши намерения.

Это как сказать строителю: 'Построй дом'. Какой дом? Сколько этажей? Из чего? Где? Модели нужна такая же точность."

## СЛАЙД 3: Анатомия промпта (4 компонента)

```
┌─────────────────────────────────────────┐
│  АНАТОМИЯ ЭФФЕКТИВНОГО ПРОМПТА          │
├─────────────────────────────────────────┤
│                                         │
│  1. РОЛЬ (Кто ты?)                      │
│     └─ Ты эксперт по контент-маркетингу│
│                                         │
│  2. КОНТЕКСТ (Что нужно знать?)         │
│     └─ Аудитория: маркетологи B2B       │
│     └─ Цель: привлечь на вебинар        │
│                                         │
│  3. ИНСТРУКЦИЯ (Что делать?)            │
│     └─ Напиши статью на 1000 слов       │
│     └─ Тема: email-маркетинг в 2024     │
│                                         │
│  4. ФОРМАТ (Как выдать результат?)      │
│     └─ Структура: введение, 5 советов, │
│        призыв к действию                │
│                                         │
└─────────────────────────────────────────┘
```

**Что говорить:**
"Запомните эти 4 компонента. Это основа любого промпта:

**1. РОЛЬ** – кем должна быть модель? Эксперт, учитель, консультант?
Почему важно: модель меняет стиль и глубину ответа в зависимости от роли.

**2. КОНТЕКСТ** – что модель должна знать о ситуации?
Почему важно: без контекста модель додумывает сама, и часто неправильно.

**3. ИНСТРУКЦИЯ** – что конкретно нужно сделать?
Почему важно: это ядро промпта. Чем точнее, тем лучше результат.

**4. ФОРМАТ** – как должен выглядеть ответ?
Почему важно: модель может выдать текст, список, таблицу, JSON – нужно указать.

**Правило:** Чем больше компонентов вы используете, тем лучше результат."

## СЛАЙД 4: Хороший промпт (пример)

```
✅ ХОРОШИЙ ПРОМПТ:

Ты – эксперт по контент-маркетингу с 10-летним опытом в B2B.

Контекст:
- Аудитория: маркетологи в B2B-компаниях, возраст 28-40 лет
- Цель статьи: привлечь на платный вебинар по email-маркетингу
- Тон: экспертный, но дружелюбный

Задача:
Напиши статью на тему "5 стратегий email-маркетинга для B2B в 2024 году"

Формат:
- Объем: 1000-1200 слов
- Структура:
  • Введение (проблема + интрига)
  • 5 стратегий (по 150 слов каждая)
  • Призыв к действию (зарегистрироваться на вебинар)
- Стиль: без воды, с конкретными примерами
```

**Демонстрация:**
1. Скопируйте этот промпт в ChatGPT
2. Покажите результат
3. Сравните с предыдущим плохим примером

**Что говорить:**
"Видите разницу? Все 4 компонента присутствуют:
- Роль: эксперт по контент-маркетингу
- Контекст: аудитория, цель, тон
- Инструкция: тема и что писать
- Формат: структура и объем

Результат получился точным, полезным, готовым к использованию.

**Запишите:** РОЛЬ → КОНТЕКСТ → ИНСТРУКЦИЯ → ФОРМАТ. Это ваша формула на сегодня."

## СЛАЙД 6: Фреймворк RISEN

```
┌─────────────────────────────────────────┐
│  RISEN (для бизнес-задач)               │
├─────────────────────────────────────────┤
│                                         │
│  R – Role (Роль)                        │
│      Кто ты?                            │
│                                         │
│  I – Input (Входные данные)             │
│      Что у нас есть?                    │
│                                         │
│  S – Steps (Шаги)                       │
│      Что делать пошагово?               │
│                                         │
│  E – Expectation (Ожидание)             │
│      Какой результат нужен?             │
│                                         │
│  N – Narrowing (Уточнение)              │
│      Ограничения и нюансы               │
│                                         │
└─────────────────────────────────────────┘
```

**Пример RISEN:**
```
R: Ты – бизнес-аналитик с опытом в e-commerce

I: У меня есть данные продаж за последние 3 месяца:
   - Январь: 150 заказов, средний чек $75
   - Февраль: 180 заказов, средний чек $82
   - Март: 210 заказов, средний чек $79

S:
1. Проанализируй тренд продаж
2. Выяви аномалии
3. Дай 3 рекомендации по увеличению среднего чека

E: Хочу получить короткий отчет с графиками (используй ASCII-графики)

N:
- Не предлагай увеличение цен
- Фокус на upsell и cross-sell
```

**Что говорить:**
"RISEN идеален для бизнес-задач: анализ, планирование, стратегия.

**Когда использовать:**
- Нужен анализ данных
- Нужен план действий
- Работаете с числами и фактами

**Запомните логику:** Роль → Данные → Шаги → Результат → Ограничения"

## СЛАЙД 7: Фреймворк CRISPE

```
┌─────────────────────────────────────────┐
│  CRISPE (для креатива)                  │
├─────────────────────────────────────────┤
│                                         │
│  C – Capacity and Role (Роль)           │
│      Кто ты?                            │
│                                         │
│  R – Insight (Инсайт)                   │
│      Ключевая идея/проблема             │
│                                         │
│  I – Statement (Утверждение)            │
│      Что мы хотим донести?              │
│                                         │
│  S – Personality (Личность)             │
│      Каким стилем говорим?              │
│                                         │
│  P – Experiment (Эксперимент)           │
│      Попробуй разные варианты           │
│                                         │
│  E – Expectation (Ожидание)             │
│      Какой результат?                   │
│                                         │
└─────────────────────────────────────────┘
```

**Пример CRISPE:**
```
C: Ты – креативный копирайтер с опытом в рекламе товаров для дома

R: Люди не замечают рекламу пылесосов, потому что она скучная

I: Наш пылесос делает уборку быстрее на 40%, но главное –
   он очень тихий (30 дБ)

S: Стиль: остроумный, с легкой иронией, без пафоса

P: Придумай 5 разных слоганов для рекламы в Instagram

E: Каждый слоган:
   - Не длиннее 10 слов
   - Зацепляющий и запоминающийся
   - Акцент на тишину работы
```

**Что говорить:**
"CRISPE – для креативных задач: тексты, слоганы, контент.

**Когда использовать:**
- Нужны идеи и варианты
- Работаете с текстами, дизайном, маркетингом
- Важен стиль и подача

**Фишка:** Буква P (Experiment) – просите несколько вариантов. Модель покажет разные подходы."

## СЛАЙД 8: Фреймворк CREATE

```
┌─────────────────────────────────────────┐
│  CREATE (для контента)                  │
├─────────────────────────────────────────┤
│                                         │
│  C – Character (Персонаж)               │
│      От чьего лица?                     │
│                                         │
│  R – Request (Запрос)                   │
│      Что создать?                       │
│                                         │
│  E – Examples (Примеры)                 │
│      Покажи образцы                     │
│                                         │
│  A – Adjustments (Корректировки)        │
│      Что изменить в стиле?              │
│                                         │
│  T – Type of Output (Тип вывода)        │
│      Формат результата                  │
│                                         │
│  E – Extras (Дополнительно)             │
│      Особые требования                  │
│                                         │
└─────────────────────────────────────────┘
```

**Пример CREATE:**
```
C: Ты – продукт-менеджер SaaS-стартапа, пишешь для блога компании

R: Создай статью "Как выбрать CRM-систему для малого бизнеса"

E: Вот примеры нашего стиля:
   - Используем "мы" и "вы" (не "они")
   - Короткие абзацы (3-4 строки)
   - Много списков и подзаголовков

A:
   - Тон: дружелюбный эксперт, не продавец
   - Избегай: сленг, сложные термины без объяснений
   - Используй: аналогии из жизни

T: Markdown-формат, готовый к публикации

E:
   - Добавь в конце CTA: "Попробуйте нашу CRM бесплатно"
   - Вставь 2-3 места для скриншотов (обозначь как [СКРИНШОТ: описание])
   - SEO: используй ключевые слова "CRM для бизнеса" 3-5 раз
```

**Что говорить:**
"CREATE – специально для создания контента: статьи, посты, инструкции.

**Когда использовать:**
- Создаете длинный контент
- Важен единый стиль (например, корпоративный)
- Нужно показать модели примеры

**Фишка:** Буква E (Examples) – покажите модели образцы. Модель скопирует стиль."

## СЛАЙД 9: Фреймворк RTF

```
┌─────────────────────────────────────────┐
│  RTF (для быстрых задач)                │
├─────────────────────────────────────────┤
│                                         │
│  R – Role (Роль)                        │
│      Кто ты?                            │
│                                         │
│  T – Task (Задача)                      │
│      Что сделать?                       │
│                                         │
│  F – Format (Формат)                    │
│      В каком виде результат?            │
│                                         │
└─────────────────────────────────────────┘
```

**Пример RTF:**
```
R: Ты – HR-специалист

T: Составь список из 10 вопросов для собеседования на позицию
   junior front-end разработчика

F: Markdown-таблица с колонками:
   - Вопрос
   - Что проверяет (hard skill / soft skill)
   - Пример хорошего ответа
```

**Что говорить:**
"RTF – самый простой фреймворк. Когда некогда, используйте его.

**Когда использовать:**
- Быстрая задача (5-10 минут)
- Простой запрос без сложного контекста
- Нужен конкретный результат в конкретном формате

**Запомните:** RTF = минимализм. Роль, задача, формат – и готово."

## СЛАЙД 11: Токены и ограничения

```
╔════════════════════════════════════════╗
║  ТОКЕНЫ: ЧТО ЭТО И КАК С НИМИ ЖИТЬ    ║
╠════════════════════════════════════════╣
║                                        ║
║  Токен ≈ 0.75 слова (на английском)    ║
║  Токен ≈ 0.5 слова (на русском)        ║
║                                        ║
║  Примеры:                              ║
║  • "Привет" = 1 токен                  ║
║  • "Искусственный интеллект" = 4       ║
║                                        ║
╚════════════════════════════════════════╝

Лимиты бесплатных версий (2024):
┌────────────────────────────────────────┐
│  ChatGPT (GPT-4):                      │
│  └─ 40 сообщений / 3 часа              │
│  └─ ~8000 токенов на ответ             │
│                                        │
│  Claude (бесплатный):                  │
│  └─ ~50 сообщений / 4 часа             │
│  └─ ~100к токенов на запрос            │
│                                        │
│  Gemini (бесплатный):                  │
│  └─ ~60 запросов / минута              │
│  └─ 32к токенов контекста              │
└────────────────────────────────────────┘
```

**Что говорить:**
"Токены – это единицы текста, которые модель обрабатывает. Как мегабайты для файлов.

**Почему это важно:**
1. Бесплатные версии ограничены по токенам
2. Длинный промпт = больше токенов = быстрее закончится лимит

**Как экономить токены:**
- Убирайте лишние слова из промпта
- Не копируйте огромные тексты в контекст
- Разбивайте большие задачи на части

**Пример:**

❌ Плохо (много токенов):
'Я работаю в компании, которая занимается разработкой программного обеспечения для малого и среднего бизнеса, и мне нужно написать письмо...'

✅ Хорошо (мало токенов):
'Контекст: разработка ПО для B2B. Задача: написать письмо...'

**Совет:** Пишите промпты коротко, но точно."

## СЛАЙД 12: Structured Outputs (JSON mode)

```
╔════════════════════════════════════════╗
║  СТРУКТУРИРОВАННЫЙ ВЫВОД               ║
╠════════════════════════════════════════╣
║                                        ║
║  Форматы:                              ║
║  • JSON – для данных, API              ║
║  • XML – для конфигураций              ║
║  • Markdown – для документов           ║
║  • CSV – для таблиц                    ║
║                                        ║
╚════════════════════════════════════════╝
```

**Пример: JSON-вывод**
```
Промпт:
---
Проанализируй этот отзыв: "Товар пришел быстро, но упаковка была помята.
Качество хорошее, но цена высокая."

Выдай результат в JSON:
{
  "sentiment": "positive/neutral/negative",
  "rating": 1-5,
  "pros": ["список"],
  "cons": ["список"],
  "topics": ["список тем"]
}
---

Результат:
{
  "sentiment": "neutral",
  "rating": 3,
  "pros": ["Быстрая доставка", "Хорошее качество"],
  "cons": ["Поврежденная упаковка", "Высокая цена"],
  "topics": ["доставка", "упаковка", "качество", "цена"]
}
```

**Что говорить:**
"Структурированный вывод – это когда модель выдает результат в формате, который можно сразу использовать.

**Зачем это нужно:**
- Можно обработать результат кодом
- Легко парсить и анализировать
- Стандартизация: всегда один формат

**Где использовать:**
- Анализ отзывов → JSON
- Генерация документации → Markdown
- Экспорт данных → CSV
- Конфигурационные файлы → XML

**Важно:** Всегда показывайте модели пример структуры – она скопирует формат."

## СЛАЙД 13: Практические советы

```
╔════════════════════════════════════════╗
║  7 ПРАВИЛ ЭФФЕКТИВНЫХ ПРОМПТОВ         ║
╠════════════════════════════════════════╣
║                                        ║
║  1. Будьте специфичны                  ║
║     ❌ "Напиши статью"                 ║
║     ✅ "Напиши статью на 800 слов"     ║
║                                        ║
║  2. Используйте примеры                ║
║     Покажите модели, что хотите        ║
║                                        ║
║  3. Разбивайте сложные задачи          ║
║     Лучше 3 промпта, чем 1 огромный    ║
║                                        ║
║  4. Указывайте формат                  ║
║     Markdown, JSON, список, таблица    ║
║                                        ║
║  5. Итерируйте                         ║
║     Первый результат – редко идеален   ║
║                                        ║
║  6. Тестируйте на разных моделях       ║
║     ChatGPT vs Claude = разные стили   ║
║                                        ║
║  7. Создавайте библиотеку шаблонов     ║
║     Сохраняйте успешные промпты        ║
║                                        ║
╚════════════════════════════════════════╝
```

**Что говорить:**
"Запишите эти 7 правил. Это чек-лист для проверки ваших промптов.
