Проблема: Почему промпты не работают?

Типичный промпт новичка:

Напиши статью про маркетинг

Что получаем:

  • Общий, размытый текст
  • Не под нашу аудиторию
  • Без конкретики и примеров
  • Модель угадывает наши намерения

Промпт = рецепт. Без деталей повар приготовит "что-то".

4 компонента эффективного промпта

1. РОЛЬ (Who)

Кем должна быть модель?

Пример: "Ты - эксперт по контент-маркетингу"

2. КОНТЕКСТ (What)

Что модель должна знать?

Пример: "Аудитория: маркетологи B2B"

3. ИНСТРУКЦИЯ (How)

Что конкретно делать?

Пример: "Напиши статью на 1000 слов"

4. ФОРМАТ (Output)

Как должен выглядеть результат?

Пример: "JSON с полями..."

РОЛЬ -> КОНТЕКСТ -> ИНСТРУКЦИЯ -> ФОРМАТ

Хороший промпт: все 4 компонента

Ты - эксперт по контент-маркетингу с 10-летним опытом в B2B. Контекст: - Аудитория: маркетологи в B2B-компаниях, возраст 28-40 лет - Цель статьи: привлечь на платный вебинар по email-маркетингу - Тон: экспертный, но дружелюбный Задача: Напиши статью на тему "5 стратегий email-маркетинга для B2B в 2024 году" Формат: - Объем: 1000-1200 слов - Структура: введение + 5 стратегий + призыв к действию - Стиль: без воды, с конкретными примерами

Результат: Качественная статья точно по ТЗ

4 фреймворка промптинга

Фреймворки - это готовые структуры. Как рецепты: заполнил пункты - получил результат.

Фреймворк Для чего Когда использовать
RISEN Бизнес-задачи Анализ, планы, отчеты
CRISPE Креатив Слоганы, идеи, концепции
CREATE Контент Статьи, гайды, документация
RTF Быстрые задачи Списки, шаблоны, простые тексты

Фреймворк RISEN - для бизнес-задач

R
Role - Кто ты? (роль эксперта)
I
Input - Входные данные (что у нас есть)
S
Steps - Шаги (что делать пошагово)
E
Expectation - Ожидание (какой результат)
N
Narrowing - Уточнение (ограничения)

Идеально для: анализа данных, планирования, стратегии, работы с числами

Фреймворк CRISPE - для креатива

C
Capacity - Роль (кто ты)
R
Insight - Инсайт (ключевая идея/проблема)
I
Statement - Утверждение (что донести)
S
Personality - Стиль (каким тоном)
P
Experiment - Варианты (попробуй разные)
E
Expectation - Результат

Фишка: Буква P - просите несколько вариантов, модель покажет разные подходы

Фреймворк CREATE - для контента

C
Character - Персонаж (от чьего лица)
R
Request - Запрос (что создать)
E
Examples - Примеры (образцы стиля)
A
Adjustments - Корректировки (тон, стиль)
T
Type - Тип вывода (формат)
E
Extras - Дополнительно (особые требования)

Фишка: Примеры (E) - покажите модели образцы, она скопирует стиль!

Фреймворк RTF - для быстрых задач

R
Role - Кто ты?
T
Task - Что сделать?
F
Format - В каком виде результат?
R: Ты - HR-специалист T: Составь список из 10 вопросов для собеседования на позицию junior front-end разработчика F: Markdown-таблица: Вопрос | Тип | Пример ответа

RTF = минимализм. Когда времени мало - это ваш выбор.

Токены и ограничения

Что такое токен?

  • Токен ~ 0.75 слова (англ.)
  • Токен ~ 0.5 слова (рус.)
  • "Привет" = 1 токен
  • "Искусственный интеллект" = 4 токена

Лимиты (2024)

  • ChatGPT: 40 сообщ./3 часа
  • Claude: ~50 сообщ./4 часа
  • Gemini: ~60 запросов/мин
Как экономить токены: Убирайте лишние слова, не копируйте огромные тексты, разбивайте большие задачи на части

Структурированный вывод (JSON)

Когда результат можно сразу обработать кодом

Промпт: Проанализируй отзыв: "Товар хороший, но доставка медленная. Цена приемлемая." Выдай в JSON: { "sentiment": "positive/neutral/negative", "rating": 1-5, "pros": ["список"], "cons": ["список"] } Результат: { "sentiment": "neutral", "rating": 3, "pros": ["Хорошее качество", "Приемлемая цена"], "cons": ["Медленная доставка"] }

Где использовать: анализ отзывов, классификация, автоматизация, API

7 правил эффективных промптов

1Будьте специфичны - "1000 слов" вместо "статью"
2Используйте примеры - покажите модели, что хотите
3Разбивайте сложное - лучше 3 промпта, чем 1 огромный
4Указывайте формат - Markdown, JSON, таблица
5Итерируйте - первый результат редко идеален
6Тестируйте на разных моделях - ChatGPT vs Claude
7Создавайте библиотеку - сохраняйте успешные промпты
