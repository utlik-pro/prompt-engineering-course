Продвинутый промптинг в Cursor

.cursorrules, цепочки промптов, агентский режим и персональная библиотека

Заметка для преподавателя

Цель урока: Перевести студентов на продвинутый уровень работы с Cursor. Это ключевой урок модуля -- здесь промпт-инженерия раскрывается в полную силу.

Важно: Каждая техника должна быть продемонстрирована на живом примере. Создайте .cursorrules прямо на занятии, покажите разницу "до" и "после".

Практика: Студенты должны уйти с урока с настроенным .cursorrules, одной готовой цепочкой промптов и началом своей промпт-библиотеки.

.cursorrules -- системный промпт для вашего проекта

.cursorrules -- это файл в корне проекта, который определяет поведение ИИ по умолчанию. Это ваш системный промпт, который автоматически добавляется к каждому запросу.

Без .cursorrules

Каждый раз объясняете контекст заново. ИИ не знает ваших предпочтений, стиль общения, структуру проекта.

С .cursorrules

ИИ сразу знает: кто вы, как отвечать, какой формат использовать, какие файлы важны. Экономия 50% времени.

Как создать .cursorrules

Создайте файл .cursorrules в корне вашего проекта. Вот пример для системы управления жизнью:

.cursorrules 
# Контекст проекта
Это моя персональная система управления жизнью и продуктивностью.
Проект содержит планы, задачи, заметки, бюджет и цели.

# Обо мне
- Имя: [Ваше имя]
- Роль: промпт-инженер, менеджер проектов
- Город: Минск
- Язык общения: русский

# Правила ответов
- Отвечай на русском языке
- Используй Markdown-форматирование
- Будь конкретным и практичным
- Не используй эмодзи в документах (только в чате)
- При создании файлов используй дату в формате YYYY-MM-DD

# Структура проекта
- /plans -- планы (дневные, недельные, месячные)
- /tasks -- задачи и проекты
- /notes -- заметки и идеи
- /finance -- бюджет и финансы
- /templates -- шаблоны документов
- /digests -- дайджесты и отчёты
- /prompts -- библиотека промптов

# Предпочтения
- Для задач используй формат: [ ] задача -- @контекст
- Приоритеты: P1 (критично), P2 (важно), P3 (можно отложить)
- Для дат используй формат: 2026-03-12
- Всегда добавляй дату создания в начало документа
Проверка: После создания файла задайте в Chat простой вопрос: "Создай план на завтра". ИИ должен автоматически использовать вашу структуру папок, формат дат и стиль.
Цепочки промптов (Multi-step workflows)

Цепочка промптов -- это последовательность запросов, где результат одного шага становится входом для следующего. Это мощнейшая техника для сложных задач.

Пример: Еженедельное планирование

1
Шаг 1: Анализ 
Проанализируй файлы в /plans/week-10/:
- Какие задачи выполнены?
- Какие перенесены?
- Какие цели достигнуты?
Создай краткое резюме в /reviews/week-10-review.md
2
Шаг 2: Планирование 
На основе @reviews/week-10-review.md
и файла @goals/q1-2026.md:
- Определи 3 главные цели на неделю 11
- Распредели задачи по дням
- Учти перенесённые задачи из прошлой недели
Сохрани в /plans/week-11/plan.md
3
Шаг 3: Действие 
На основе @plans/week-11/plan.md
создай детальный план на понедельник:
- Разбей каждую задачу на подзадачи (15-30 мин)
- Добавь тайм-блоки с 9:00 до 18:00
- Оставь буфер 30 мин после обеда
Сохрани в /plans/week-11/monday.md
Принцип: Анализ -> Планирование -> Действие. Каждый шаг использует результат предыдущего через ссылки @file.
Управление контекстом

Контекстное окно -- это объём информации, который ИИ может "видеть" одновременно. В Cursor есть мощные инструменты для управления контекстом:

Команда Что делает Пример
@file Подключает конкретный файл @plans/today.md проанализируй план
@folder Подключает все файлы папки @tasks/ покажи статистику задач
@web Ищет в интернете @web лучшие практики GTD 2026
@codebase Ищет по всему проекту @codebase найди все файлы с OKR
@docs Подключает документацию @docs как использовать Notion API

Стратегии работы с контекстом:

Точечный контекст

Подключайте только нужные файлы: @tasks/active.md @goals/march.md. Чем точнее контекст, тем лучше ответ.

Широкий контекст

Используйте @folder или @codebase, когда нужен обзор всего проекта или поиск по нескольким файлам.

Внешний контекст

Добавляйте @web, когда нужна актуальная информация из интернета для принятия решений.

Few-shot в Cursor: учим ИИ через примеры

Few-shot -- техника, когда вы даёте ИИ несколько примеров нужного формата, и он повторяет паттерн. В Cursor это делается через файлы-примеры.

Пример: формат ежедневного отчёта

templates/daily-report-example.md 
# Ежедневный отчёт -- 2026-03-10

## Выполнено сегодня
- [x] Подготовить презентацию для клиента -- @работа P1
- [x] Оплатить счета за интернет -- @быт P2
- [ ] Прочитать главу книги -- @развитие P3 (перенос)

## Ключевой результат дня
Презентация готова и отправлена клиенту. Получил
положительный отзыв.

## На завтра
- [ ] Созвон с командой в 10:00 -- @работа P1
- [ ] Тренировка в 18:00 -- @здоровье P2

## Заметки
Идея: автоматизировать отчёты через Cursor.
Промпт, использующий пример 
Создай ежедневный отчёт за сегодня (2026-03-12).
Используй точно такой же формат, как в
@templates/daily-report-example.md.

Мои результаты за сегодня:
- Закончил урок по интеграциям в Cursor
- Настроил Telegram-бота для напоминаний
- Не успел сделать бюджетный отчёт
- Завтра: встреча с коллегами, подготовка проекта
Результат: ИИ создаст отчёт в точно таком же формате -- с чекбоксами, тегами @контекст, приоритетами P1-P3. Один пример стоит тысячи слов описания!
Chain of Thought: пошаговое мышление

CoT (Chain of Thought) -- техника, которая заставляет ИИ думать пошагово. Особенно полезна для сложного анализа и принятия решений.

Без CoT (простой промпт):

Слабый промпт 
Стоит ли мне менять работу?

С CoT (структурированный промпт):

Сильный промпт с CoT 
Помоги мне проанализировать решение о смене работы.
Думай пошагово:

Шаг 1: Проанализируй мою текущую ситуацию
(см. @notes/career-current.md)

Шаг 2: Оцени предложение новой компании
(см. @notes/job-offer.md)

Шаг 3: Сравни по критериям:
- Зарплата и рост
- Интерес к задачам
- Work-life balance
- Перспективы через 2 года

Шаг 4: Перечисли риски каждого варианта

Шаг 5: Дай рекомендацию с обоснованием

Сохрани анализ в decisions/job-change-analysis.md
Когда использовать
  • Сложные решения
  • Анализ данных
  • Стратегическое планирование
  • Оценка рисков
Ключевые фразы
  • "Думай пошагово"
  • "Сначала проанализируй, потом..."
  • "Шаг 1... Шаг 2... Шаг 3..."
  • "Обоснуй каждый вывод"
Агентский режим Cursor

Cursor Agent -- это режим, в котором ИИ может автономно выполнять задачи: создавать файлы, запускать команды в терминале, читать результаты и продолжать работу.

Chat vs Agent

Chat:

Отвечает на вопросы, предлагает код. Вы сами применяете изменения.

Agent:

Сам создаёт файлы, запускает скрипты, исправляет ошибки. Работает автономно.

Пример задачи для Agent:

Промпт для Agent (Composer) 
Создай систему утреннего обзора:

1. Создай папку morning-routine/ если её нет
2. Создай шаблон morning-routine/template.md:
   - Секция "Погода" (заглушка)
   - Секция "Задачи на сегодня" (из @tasks/active.md)
   - Секция "Расписание"
   - Секция "Фокус дня"
3. Создай скрипт morning-routine/generate.py, который:
   - Копирует шаблон с текущей датой
   - Подставляет задачи из tasks/active.md
   - Сохраняет в morning-routine/2026-03-12.md
4. Запусти скрипт и проверь результат
5. Если есть ошибки -- исправь и запусти снова
Важно: В агентском режиме всегда проверяйте, что делает ИИ. Cursor спросит подтверждение перед выполнением каждой команды в терминале. Не нажимайте "Да" бездумно!
Промпт-библиотека

Со временем у вас накопятся промпты, которые работают отлично. Организуйте их в библиотеку -- это ваша суперсила.

Структура библиотеки:

Структура папки prompts/ 
prompts/
├── planning/
│   ├── daily-plan.md          # Создание плана на день
│   ├── weekly-review.md       # Еженедельный обзор
│   └── monthly-retro.md       # Месячная ретроспектива
├── analysis/
│   ├── decision-matrix.md     # Анализ решений
│   ├── budget-review.md       # Анализ бюджета
│   └── goal-progress.md       # Прогресс по целям
├── content/
│   ├── meeting-summary.md     # Резюме встречи
│   ├── email-draft.md         # Черновик письма
│   └── presentation-outline.md # План презентации
└── automation/
    ├── digest-creator.md      # Создание дайджеста
    ├── data-processor.md      # Обработка данных
    └── report-generator.md    # Генерация отчётов

Формат промпта в библиотеке:

prompts/planning/daily-plan.md 
# Промпт: Создание плана на день
Версия: 1.2
Последнее обновление: 2026-03-10

## Когда использовать
Каждое утро для создания структурированного плана дня.

## Промпт
---
Создай план на сегодня ({{ДАТА}}).

Входные данные:
- Активные задачи: @tasks/active.md
- Цели недели: @plans/current-week.md
- Календарь: @calendar/today.md (если есть)

Формат:
1. Фокус дня (1 главная задача)
2. Расписание по блокам (9:00-18:00)
3. Список "если останется время"
4. Вечерний чекпоинт

Правила:
- Не более 5 задач в день
- Блоки по 90 минут с перерывами 15 мин
- Обед 12:30-13:30
---

## Примечания
- Работает лучше с .cursorrules настроенным
- Можно добавить @web для учёта погоды
Как использовать: Откройте промпт из библиотеки, скопируйте секцию "Промпт" в Chat, подставьте сегодняшнюю дату. За 10 секунд -- готовый план!

Для хранения паролей проекта
Для настройки цветовой темы Cursor

Практическое задание

Выполните три задания:

Задание 1: Настройте .cursorrules

  1. Создайте файл .cursorrules в корне вашего проекта
  2. Опишите себя, свой проект, предпочтения по формату
  3. Задайте структуру папок и правила именования файлов
  4. Проверьте: задайте ИИ вопрос и убедитесь, что правила работают

Задание 2: Создайте цепочку промптов

  1. Выберите сложную задачу (планирование недели, анализ бюджета)
  2. Разбейте на 3 шага: анализ -> план -> действие
  3. Выполните каждый шаг последовательно, ссылаясь на результаты предыдущего

Задание 3: Начните промпт-библиотеку

  1. Создайте папку prompts/ с подпапками по категориям
  2. Сохраните минимум 3 промпта, которые вы уже использовали в курсе
  3. Для каждого промпта укажите: когда использовать, сам промпт, примечания
