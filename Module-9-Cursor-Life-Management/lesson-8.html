Работа с файлами и данными

Cursor как персональный аналитик: обработка CSV, JSON и текстов без Excel

Заметка для преподавателя

Цель урока: Показать, что Cursor может заменить Excel для анализа данных, обработки текстов и генерации отчётов. Никакого программирования — только промпты.

Ключевой момент: Студенты часто не осознают, что текстовый редактор с ИИ может быть мощнее электронных таблиц для многих задач. Покажите это на конкретных примерах.

Подготовка: Подготовьте пример CSV-файла с расходами за месяц (10-15 строк). Студенты будут работать с ним на практике.

Cursor как аналитик данных

Зачем открывать Excel, если у вас есть Cursor? ИИ-ассистент может анализировать данные, находить закономерности и строить отчёты — всё через естественный язык.

Традиционный путь
1. Открыть Excel
2. Импортировать данные
3. Написать формулы
4. Создать графики
5. Собрать отчёт
Время: 30-60 минут
Путь через Cursor
1. Открыть файл в Cursor
2. Спросить ИИ: "проанализируй"
3. Получить отчёт


Время: 2-5 минут
Анализ CSV-файлов

CSV (Comma-Separated Values) — самый популярный формат для данных. Cursor читает его и анализирует через ИИ.

Пример: анализ бюджета

budget.csv 
Дата,Категория,Описание,Сумма
2026-03-01,Продукты,Еженедельная закупка,-85.50
2026-03-01,Транспорт,Метро на месяц,-45.00
2026-03-02,Развлечения,Кино с друзьями,-22.00
2026-03-03,Продукты,Кофе и снеки,-12.30
2026-03-04,Подписки,Spotify,-9.99
2026-03-05,Зарплата,Основная работа,+2500.00
2026-03-06,Продукты,Доставка еды,-35.00
2026-03-07,Одежда,Новые кроссовки,-120.00
2026-03-08,Здоровье,Аптека,-28.50
2026-03-10,Подписки,Netflix,-15.99
2026-03-12,Продукты,Еженедельная закупка,-92.00
2026-03-15,Фриланс,Дизайн логотипа,+350.00

Что спросить у ИИ в Chat (Cmd+L):

"Проанализируй мои расходы из @budget.csv. Покажи общую сумму по категориям, процент от доходов, и дай рекомендации по оптимизации"
"Какая категория расходов самая большая? Покажи топ-3 категории и средний чек в каждой"
"Создай Markdown-таблицу с итогами по каждой категории: количество транзакций, общая сумма, средняя сумма"
Синтаксис @file: Используйте @имя_файла в Chat, чтобы подключить файл к контексту. ИИ прочитает его содержимое и сможет анализировать данные.
Массовая обработка текстов

Cursor идеален для обработки больших объёмов текстовых данных: резюме, отзывов, заметок, протоколов встреч.

Пример 1: Обработка резюме

У вас 10 резюме кандидатов в текстовом формате. Нужно быстро их сравнить.

Промпт для Chat
Проанализируй резюме из файлов @resume-1.md @resume-2.md @resume-3.md

Создай сравнительную таблицу:
- Имя
- Опыт работы (лет)
- Ключевые навыки
- Образование
- Рекомендация (подходит/не подходит) для позиции маркетолога

Пример 2: Анализ отзывов

Файл с 50 отзывами клиентов. Нужно понять общее настроение.

Промпт для Chat
Проанализируй отзывы из @reviews.txt

1. Раздели на позитивные, нейтральные и негативные
2. Выдели топ-5 самых частых жалоб
3. Выдели топ-5 похвал
4. Дай общую оценку и рекомендации по улучшению

Пример 3: Протоколы встреч

Длинные заметки с совещания. Нужно извлечь решения и задачи.

Промпт для Chat
Из заметок встречи @meeting-notes.md извлеки:

1. Принятые решения (пронумерованный список)
2. Задачи с ответственными и дедлайнами (таблица)
3. Открытые вопросы, требующие обсуждения
4. Краткое резюме встречи (3-4 предложения)
Конвертация форматов

Cursor мгновенно конвертирует данные между форматами. Выделяете текст, используете Cmd+K или Chat — получаете нужный формат.

Из формата В формат Команда Cmd+K
Markdown HTML "конвертировать в HTML"
CSV Markdown-таблица "превратить в красивую Markdown-таблицу"
JSON Читаемый текст "объяснить эти данные простым языком"
Список CSV "конвертировать в CSV с заголовками"
Таблица JSON "преобразовать в JSON-массив объектов"
Текст Markdown "оформить как Markdown с заголовками и списками"
Пример конвертации JSON в Markdown:
JSON (до)
[
  {"name": "Иван", "role": "PM", "tasks": 5},
  {"name": "Мария", "role": "Дизайнер", "tasks": 3},
  {"name": "Алекс", "role": "Разработчик", "tasks": 8}
]
Markdown (после)
| Имя   | Роль        | Задачи |
|-------|-------------|--------|
| Иван  | PM          | 5      |
| Мария | Дизайнер    | 3      |
| Алекс | Разработчик | 8      |
Работа с большими файлами через @file

Cursor позволяет подключать файлы к контексту ИИ через специальный синтаксис @. Это ключевая функция для анализа данных.

1
Откройте Chat (Cmd+L)

Или Composer (Cmd+I) для создания новых файлов на основе данных

2
Введите @ и начните печатать имя файла

Появится выпадающий список с подходящими файлами из проекта

3
Выберите файл и напишите запрос

ИИ прочитает содержимое файла и будет работать с ним

@file — подключить конкретный файл

Пример: @budget.csv проанализируй расходы

@folder — подключить папку целиком

Пример: @data/ найди аномалии во всех файлах

Несколько файлов — подключить сразу несколько

Пример: @january.csv @february.csv сравни расходы за два месяца

Ограничение: ИИ имеет лимит на размер контекста. Если файл очень большой (тысячи строк), лучше разбить анализ на части или попросить ИИ обработать первые N строк.
Извлечение информации из неструктурированного текста

Одна из суперспособностей ИИ — извлекать структурированные данные из хаотичного текста. Вот типичные сценарии:

Из текста письма — в задачи

Исходный текст
Привет! Обсудили с клиентом проект. Нужно до пятницы
отправить ему макеты. Ещё он просил обновить логотип
и написать тексты для лендинга. Бюджет утвердили —
150 000 руб. Встреча в понедельник в 14:00.

Промпт: "Извлеки из этого текста: задачи, дедлайны, суммы, даты встреч. Оформи как структурированный список."

Из визитки — в контакт

Исходные данные
Иванов Петр Сергеевич
Директор по маркетингу
ООО "ТехноСтарт"
+7 (999) 123-45-67
p.ivanov@technostart.ru
г. Москва, ул. Пушкина, д. 10

Промпт: "Конвертировать в JSON-формат контакта: имя, должность, компания, телефон, email, адрес."

Из чека — в таблицу

Текст чека из магазина или скопированные данные из приложения банка — ИИ разберёт строки и создаст аккуратную таблицу с категориями.

Промпт: "Разбери этот чек: извлеки название товара, количество, цену за единицу, итого. Оформи как Markdown-таблицу. Добавь строку с общей суммой."

Генерация отчётов

Cursor может создавать полноценные отчёты на основе ваших данных. Вот пошаговый процесс:

1
Подготовьте данные

Положите CSV, JSON или текстовые файлы в папку проекта

2
Откройте Composer (Cmd+I)

Composer создаст новый файл с отчётом

3
Дайте детальный промпт

Опишите структуру отчёта, какие данные включить, формат

4
Получите готовый отчёт

ИИ создаст файл отчёта в Markdown с таблицами, выводами и рекомендациями

Пример промпта для отчёта 
На основе данных из @budget.csv создай файл report-march.md

Структура отчёта:
# Финансовый отчёт за март 2026

## Сводка
- Общий доход
- Общие расходы
- Баланс (доходы минус расходы)

## Расходы по категориям
Таблица: категория, сумма, % от общих расходов

## Топ-3 крупнейших трат
Список с датами и описаниями

## Рекомендации по оптимизации
3-5 конкретных советов на основе данных

## Прогноз на апрель
Если тренд сохранится, какие расходы ожидать
Лайфхак: Создайте шаблон отчёта один раз. В следующем месяце просто подставите новые данные и получите отчёт за минуту.
Практика: полный цикл работы с данными

Выполните все три задания последовательно. Каждое занимает 10-15 минут.

Задание 1: Анализ CSV с расходами

  1. Создайте файл my-expenses.csv с вашими реальными или выдуманными расходами за неделю (10-15 записей)
  2. Используйте формат: Дата, Категория, Описание, Сумма
  3. Через Chat: @my-expenses.csv проанализируй мои расходы, покажи итоги по категориям
  4. Попросите ИИ найти, где можно сэкономить

Задание 2: Создание отчёта

  1. Откройте Composer (Cmd+I)
  2. Попросите: "На основе @my-expenses.csv создай файл weekly-report.md с полным финансовым отчётом"
  3. Отчёт должен включать: сводку, таблицу по категориям, рекомендации
  4. Отредактируйте отчёт через Cmd+K если нужно

Задание 3: Конвертация данных

  1. Выделите таблицу из отчёта
  2. Cmd+K → "конвертировать в CSV-формат"
  3. Создайте новый файл, вставьте CSV
  4. Выделите CSV → Cmd+K → "конвертировать в JSON-массив"
  5. Сравните три формата: Markdown, CSV, JSON
