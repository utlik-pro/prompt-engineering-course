Построение персональной системы продуктивности

Объединяем все инструменты в единую систему управления жизнью

Заметка для преподавателя

Цель урока: Собрать все навыки модуля в единую работающую систему. Это урок-синтез, где каждый студент создаёт персональную систему продуктивности в Cursor.

Важно: Урок должен быть максимально практическим. 70% времени -- самостоятельная работа студентов над своей системой. Преподаватель консультирует индивидуально.

Результат: К концу урока у каждого студента должна быть работающая система с утренним ритуалом, еженедельным обзором и дашбордом.

Объединяем всё вместе

За 11 уроков вы освоили мощный набор инструментов. Теперь пора собрать их в единую систему, которая будет работать на вас каждый день.

Chat + Composer

Создание и обработка документов, анализ, планирование

Шаблоны + Промпты

Стандартизация процессов, библиотека решений

Интеграции + Agent

Внешние данные, автоматизация, автономная работа

Утренний ритуал

Утренний ритуал -- это первое, что вы делаете каждый рабочий день. 5-10 минут в Cursor, которые задают тон всему дню.

Пошаговый шаблон утреннего ритуала:

1
Откройте Cursor и проект

1 минута. Откройте my-life-system. Глаза на dashboard.md.

2
Обзор задач

2 минуты. В Chat: "Покажи мои активные задачи из @tasks/active.md, отсортируй по приоритету"

3
Создание плана дня

3 минуты. Используйте промпт из библиотеки: @prompts/planning/daily-plan.md

4
Определение фокуса

1 минута. Выберите ОДНУ главную задачу дня. Запишите в начало плана.

5
Приступайте к работе!

План готов. Начинайте с фокус-задачи. Cursor открыт и готов помогать.

Промпт для утреннего ритуала (сохраните!) 
Доброе утро! Помоги мне начать день.

1. Посмотри @tasks/active.md -- какие задачи на сегодня?
2. Проверь @plans/weekly/current.md -- что запланировано?
3. Создай план на сегодня ({{ДАТА}}):
   - Фокус дня: 1 главная задача
   - Расписание: блоки по 90 мин, 9:00-18:00
   - Обед: 12:30-13:30
   - Максимум 5 задач
4. Сохрани в plans/daily/{{ДАТА}}.md

Если есть перенесённые задачи с вчера -- включи их.
Еженедельный обзор

Раз в неделю (лучше в пятницу вечером или воскресенье) проводите обзор недели. Это занимает 20-30 минут и кардинально повышает продуктивность.

Промпт для еженедельного обзора 
Проведи еженедельный обзор (неделя {{НОМЕР}}, {{ДАТА}}).

Шаг 1 -- РЕВЬЮ:
Проанализируй @plans/daily/ за эту неделю:
- Сколько задач выполнено / перенесено / отменено?
- Какие дни были самыми продуктивными?
- Что мешало (блокеры, отвлечения)?

Шаг 2 -- ПРОГРЕСС:
Сравни с @goals/q1-2026.md:
- Какие цели продвинулись?
- Какие отстают?

Шаг 3 -- ПЛАНИРОВАНИЕ:
На основе анализа:
- 3 главные цели на следующую неделю
- Задачи для каждой цели
- Что нужно делегировать или отменить

Шаг 4 -- УРОКИ:
- Что сработало хорошо?
- Что нужно изменить?
- Одно улучшение для процесса

Сохрани в reviews/weekly/week-{{НОМЕР}}.md
Обнови @tasks/active.md с новыми задачами
Вход
  • Дневные планы за неделю
  • Квартальные цели
  • Заметки и идеи
Выход
  • Обзор-отчёт за неделю
  • План на следующую неделю
  • Обновлённый список задач
Месячная ретроспектива

В конце каждого месяца -- более глубокий анализ. 30-45 минут для оценки прогресса и корректировки курса.

Промпт для месячной ретроспективы 
Проведи ретроспективу за {{МЕСЯЦ}} {{ГОД}}.

Данные:
- Еженедельные обзоры: @reviews/weekly/
- Финансы: @finance/expenses/{{МЕСЯЦ}}.md
- Цели: @goals/q1-2026.md
- Привычки: @goals/habits.md

Анализ:
1. ДОСТИЖЕНИЯ -- что удалось в этом месяце?
2. МЕТРИКИ:
   - Процент выполненных задач
   - Финансовый итог (доходы vs расходы)
   - Прогресс по OKR (% от цели)
3. ПРОБЛЕМЫ -- что не сработало и почему?
4. ПРИВЫЧКИ -- какие укрепились, какие провалились?
5. КОРРЕКТИРОВКА -- что изменить в следующем месяце?
6. OKR ОБНОВЛЕНИЕ -- нужно ли пересмотреть цели?

Формат: подробный отчёт с числами и графиками (ASCII).
Сохрани в reviews/monthly/{{МЕСЯЦ}}-{{ГОД}}.md
Сила накопления: Через 3 месяца у вас будет потрясающая база данных о себе. ИИ сможет находить паттерны: "Вы наиболее продуктивны по вторникам", "Расходы на кафе выросли на 30%".

Эволюция системы

Ваша система -- живой организм. Она должна расти и меняться вместе с вами. Вот как обеспечить её развитие:

Версионирование через Git

Зачем Git для системы жизни?

Git сохраняет историю всех изменений. Вы сможете посмотреть, как менялись ваши цели, планы и приоритеты. Это бесценный архив.

Базовые Git-команды (через терминал Cursor) 
# Инициализация (один раз)
git init
git add .
git commit -m "Начальная версия системы"

# Ежедневно (или в конце дня)
git add .
git commit -m "Обновление: 2026-03-12"

# Посмотреть историю
git log --oneline

# Посмотреть, что изменилось за месяц
git diff HEAD~30

Этапы роста системы:

Этап Что включает Когда
v1.0 -- Старт Дашборд, задачи, дневной план, .cursorrules Сегодня
v2.0 -- Ритуалы Утренний ритуал, еженедельный обзор, шаблоны Через 1 неделю
v3.0 -- Цели OKR, трекер привычек, финансы Через 2 недели
v4.0 -- Автоматизация Скрипты, интеграции, MCP-серверы Через 1 месяц
v5.0 -- Интеллект Анализ паттернов, прогнозы, рекомендации от ИИ Через 3 месяца
Главное правило: Лучшая система -- та, которую вы реально используете. Начните с малого. Добавляйте только то, что приносит пользу. Не усложняйте ради усложнения.

Частые ошибки (и как их избежать)
Ошибка: Слишком сложная система с первого дня

Решение: Начните с дашборда, задач и одного ритуала. Добавляйте по одному элементу в неделю.

Ошибка: Планирование вместо действий

Решение: Утренний ритуал -- 5 минут, не больше. Остальное время -- на выполнение задач, а не на идеальный план.

Ошибка: Не делать обзоры

Решение: Поставьте еженедельный обзор в календарь как встречу. Без обзоров система теряет смысл.

Ошибка: Забрасывать систему после недели

Решение: Минимальная версия -- открыть Cursor утром и записать 3 задачи на день. Даже это уже система.

Практическое задание: собираем систему

Создайте свою систему продуктивности и проведите первый утренний ритуал:

Часть 1: Создание системы (20 мин)

  1. Создайте проект my-life-system (или откройте существующий)
  2. Попросите Cursor Agent создать базовую структуру папок
  3. Настройте .cursorrules под себя
  4. Создайте dashboard.md с вашими реальными задачами
  5. Добавьте файл tasks/active.md с текущими задачами

Часть 2: Первый утренний ритуал (10 мин)

  1. Используйте промпт утреннего ритуала из урока
  2. Создайте план на сегодня
  3. Определите фокус дня
  4. Обновите дашборд

Часть 3: Git и сохранение (5 мин)

  1. Инициализируйте Git: git init
  2. Добавьте .gitignore с .env
  3. Сделайте первый коммит: git commit -m "v1.0 -- начало системы"
Поздравляю! Вы создали персональную систему продуктивности на базе Cursor. Используйте её каждый день, и через месяц вы не узнаете свою продуктивность.
