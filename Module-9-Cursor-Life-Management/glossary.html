Все ключевые термины модуля 9 в одном месте

А

Автоматизация (Automation)

Выполнение повторяющихся задач без ручного вмешательства. В контексте Cursor -- использование скриптов, pipeline-ов и ИИ для автоматического выполнения рутинных процессов.

Пример в Cursor: Скрипт, который каждое утро генерирует отчёт по привычкам из CSV-файла и создаёт план дня на основе календаря.

Agent Mode (Агентный режим)

Режим работы ИИ в Cursor, при котором модель может самостоятельно выполнять последовательность действий: читать файлы, запускать команды в терминале, создавать и редактировать документы.

Пример в Cursor: В Composer с Agent Mode ИИ может одним запросом создать структуру папок, заполнить шаблоны и запустить скрипт проверки.

API (Application Programming Interface)

Интерфейс для взаимодействия программ между собой. Позволяет одному приложению отправлять запросы другому и получать данные. В контексте ИИ -- способ отправить промпт и получить ответ программно.

Пример в Cursor: Через MCP можно подключить API Telegram, чтобы ИИ отправлял вам утреннее напоминание с планом дня.

В

Версионирование (Versioning)

Система отслеживания изменений в файлах. Позволяет вернуться к предыдущей версии документа, сравнить изменения и работать параллельно над разными вариантами.

Пример в Cursor: Через Git можно отслеживать все изменения в вашей системе управления жизнью и откатываться, если что-то пошло не так.

Workflow (Рабочий процесс)

Последовательность шагов для выполнения определённой задачи. В Cursor -- цепочка действий, которую вы выполняете регулярно: утренний обзор, недельное планирование, обработка входящих.

Пример в Cursor: Workflow "Утро": открыть трекер привычек, отметить вчерашний день, спросить Chat о приоритетах, обновить план дня.

Г

Git

Система контроля версий, встроенная в Cursor. Сохраняет историю всех изменений файлов и позволяет работать с ветками -- параллельными версиями проекта.

Пример в Cursor: Каждую неделю делайте коммит вашей системы: git commit -m "Неделя 12: обновил OKR, добавил 3 промпта".

GTD (Getting Things Done)

Методология управления задачами Дэвида Аллена. Суть: собрать все задачи в одно место (inbox), обработать каждую (сделать, делегировать, отложить, удалить), распределить по контекстам и регулярно пересматривать.

Пример в Cursor: Создайте папку gtd/ с файлами inbox.md, next-actions.md, waiting-for.md, someday.md. Используйте Chat для еженедельного обзора.

Д

Дашборд (Dashboard)

Сводная панель с ключевыми показателями и статусами. В текстовом формате -- Markdown-файл, который агрегирует информацию из разных частей вашей системы: прогресс по целям, привычки, финансы.

Пример в Cursor: Файл dashboard.md, который вы просите ИИ обновить каждое утро: "Обнови дашборд на основе данных из трекеров".

К

Контекстное окно (Context Window)

Объём информации, который ИИ-модель может "видеть" одновременно. Измеряется в токенах. Чем больше контекстное окно, тем больше файлов и истории диалога ИИ может учитывать при ответе.

Пример в Cursor: Если добавить в Chat файл с планами на квартал через @-упоминание, ИИ будет учитывать ваши цели при ответах на любые вопросы в этой сессии.

Cmd+K (Ctrl+K)

Горячая клавиша в Cursor для быстрого редактирования текста с помощью ИИ. Выделяете текст, нажимаете Cmd+K, описываете что изменить -- ИИ редактирует прямо на месте.

Пример в Cursor: Выделите список задач, нажмите Cmd+K: "Отсортируй по приоритету и добавь дедлайны на эту неделю".

CSV (Comma-Separated Values)

Текстовый формат для хранения табличных данных, где значения разделены запятыми. Легко читается как человеком, так и программами. Идеален для трекеров и отчётов.

Пример в Cursor: Файл finances.csv: 2026-03-01,Кафе,Еда,-450. ИИ в Chat может анализировать такой файл: "Посчитай мои расходы на еду за март".

Cursor

Редактор кода со встроенным ИИ, основанный на VS Code. В контексте этого модуля используется не для программирования, а как мощная платформа для управления жизнью, проектами и процессами через промпт-инженерию.

Пример: Откройте папку вашей системы в Cursor -- и у вас есть персональный ИИ-ассистент, который знает весь контекст вашей жизни.

.cursorrules

Специальный файл в корне проекта Cursor, содержащий инструкции для ИИ. Всё написанное в .cursorrules автоматически подгружается в каждый запрос как системный контекст.

Пример в Cursor: В .cursorrules: "Ты -- персональный ассистент Анны. Она фрилансер-дизайнер. Отвечай кратко. Используй метод GTD. Знай её клиентов: WebStudio, CoffeeShop".

М

Markdown

Простой язык разметки текста. Используется для создания структурированных документов: заголовки (#), списки (-), жирный (**текст**), ссылки. Основной формат файлов для систем управления жизнью в Cursor.

Пример в Cursor: Файл weekly-plan.md с заголовками по дням недели, списками задач и чекбоксами - [ ] задача.

MCP (Model Context Protocol)

Протокол, позволяющий подключать к Cursor внешние инструменты и сервисы. Через MCP ИИ получает доступ к API сторонних приложений: календари, мессенджеры, базы данных.

Пример в Cursor: Подключите MCP-сервер для Google Calendar, и ИИ сможет проверить ваше расписание: "Какие встречи у меня завтра?".

О

OKR (Objectives and Key Results)

Методология целеполагания, где каждая цель (Objective) подкреплена 2-5 измеримыми ключевыми результатами (Key Results). Популярна в Google, Intel и других компаниях.

Пример в Cursor: Цель: "Стать продуктивнее". KR1: "Выполнять 80% запланированных задач". KR2: "Вести трекер привычек 30 дней подряд". Попросите ИИ: "Оцени мой прогресс по OKR на основе данных трекеров".

П

Промпт (Prompt)

Текстовый запрос к ИИ. Может быть простым вопросом или сложной инструкцией с контекстом, ролью, форматом вывода и примерами. Качество промпта определяет качество ответа.

Пример в Cursor: Вместо "Составь план" напишите: "Составь план на неделю в формате Markdown-таблицы. Учти мои OKR из файла okr/q1-2026.md и незавершённые задачи из gtd/next-actions.md".

Pipeline (Конвейер)

Последовательность автоматических шагов, где выход одного шага становится входом следующего. В Cursor -- цепочка промптов и действий для обработки данных.

Пример в Cursor: Pipeline "Еженедельный отчёт": 1) прочитать трекер привычек, 2) проанализировать финансы, 3) оценить прогресс по OKR, 4) сгенерировать сводный отчёт.

Р

Рабочее пространство (Workspace)

Папка, открытая в Cursor, со всеми вложенными файлами и подпапками. ИИ в Cursor "видит" всё рабочее пространство и может обращаться к любому файлу в нём.

Пример в Cursor: Откройте папку my-life-system/ -- и ИИ получит доступ ко всем вашим проектам, трекерам и промптам одновременно.

Ретроспектива (Retrospective)

Регулярный обзор прошедшего периода (неделя, месяц) с анализом: что получилось, что нет, что изменить. Ключевой элемент любой системы продуктивности.

Пример в Cursor: Откройте шаблон rituals/monthly.md и попросите Chat: "Проведи ретроспективу месяца на основе моих трекеров и проектов. Что я сделал хорошо? Где провал?".

С

Системный промпт (System Prompt)

Инструкция, которая определяет поведение ИИ на протяжении всего диалога. В Cursor системный промпт задаётся через файл .cursorrules и автоматически применяется ко всем запросам.

Пример в Cursor: "Ты -- строгий, но справедливый коуч по продуктивности. Всегда спрашивай о дедлайнах. Отвечай на русском. Используй методологию OKR".

Second Brain (Второй мозг)

Концепция Тьяго Форте: внешняя система для хранения знаний, идей и информации. Вместо того чтобы держать всё в голове, вы "выгружаете" информацию в организованную структуру файлов.

Пример в Cursor: Ваше рабочее пространство в Cursor -- это и есть Second Brain. ИИ помогает не только хранить, но и анализировать, связывать и извлекать знания.

Composer

Мощный режим ИИ в Cursor (Cmd+I / Ctrl+I), который может создавать и редактировать несколько файлов одновременно. Идеален для масштабных изменений: создание структуры проекта, генерация набора шаблонов, рефакторинг системы.

Пример в Cursor: "Создай структуру GTD-системы: 5 файлов в папке gtd/ с заголовками, инструкциями и примерами записей в каждом".

Chat (Чат)

Боковая панель ИИ в Cursor (Cmd+L / Ctrl+L) для диалога с моделью. Можно задавать вопросы, анализировать файлы, генерировать идеи. Поддерживает @-упоминания файлов для добавления контекста.

Пример в Cursor: Откройте Chat, добавьте @finances.csv и спросите: "Какие мои три крупнейшие категории расходов в этом месяце?".

Т

Токены (Tokens)

Единицы текста, на которые ИИ разбивает входные данные. Примерно 1 токен = 4 символа на английском или 1-2 символа на русском. Ограничение по токенам определяет, сколько информации ИИ может обработать за раз.

Пример в Cursor: Если ваш .cursorrules на 500 токенов, а контекстное окно -- 128 000 токенов, у вас остаётся огромный запас для файлов и диалога.

Трекер (Tracker)

Инструмент для регулярного отслеживания показателей: привычек, расходов, настроения, прогресса по целям. В Cursor реализуется через CSV-файлы или Markdown-таблицы.

Пример в Cursor: Файл habits.csv с колонками: дата, медитация, спорт, чтение, сон. Каждый день добавляете строку, а ИИ анализирует тренды.

Терминал (Terminal)

Встроенный инструмент Cursor для выполнения команд операционной системы. Позволяет запускать скрипты, управлять файлами, работать с Git -- не выходя из редактора.

Пример в Cursor: Откройте терминал (Ctrl+`) и запустите: python scripts/weekly-report.py для генерации еженедельного отчёта.

Ф

Файловая система (File System)

Способ организации файлов и папок на компьютере. В контексте Cursor -- структура вашего рабочего пространства: иерархия папок, именование файлов, логика группировки.

Пример в Cursor: Структура projects/, trackers/, prompts/, rituals/ -- каждая папка отвечает за свою область жизни.

Few-shot (Обучение на примерах)

Техника промптинга, при которой вы даёте ИИ несколько примеров желаемого результата перед основным запросом. ИИ "учится" на примерах и воспроизводит паттерн.

Пример в Cursor: "Вот как я описываю задачи: '[ ] Позвонить клиенту @Дом (15 мин) #срочно'. Теперь преобразуй этот список в такой же формат: ...".

Ш

Шаблон (Template)

Заготовка документа с предопределённой структурой и плейсхолдерами. Экономит время на создание повторяющихся документов: планов, отчётов, обзоров.

Пример в Cursor: Шаблон rituals/morning.md с секциями: "Благодарности", "3 приоритета дня", "Расписание", "Заметки". Каждое утро копируете и заполняете через Chat.

Chain-of-Thought (Цепочка рассуждений)

Техника промптинга, при которой вы просите ИИ рассуждать пошагово. Улучшает качество ответов на сложные вопросы: анализ, планирование, принятие решений.

Пример в Cursor: "Проанализируй мои расходы за месяц пошагово: 1) найди общую сумму по категориям, 2) сравни с прошлым месяцем, 3) выдели аномалии, 4) дай рекомендации".

Дополнительные термины

JSON (JavaScript Object Notation)

Текстовый формат для хранения структурированных данных. Используется для настроек, конфигураций и обмена данными между приложениями. Поддерживает вложенные объекты и массивы.

Пример в Cursor: Файл конфигурации MCP-серверов .mcp.json или структурированное хранение контактов клиентов: {"name": "WebStudio", "contact": "anna@web.com", "budget": 5000}.

Zettelkasten (Метод карточек)

Система управления знаниями, разработанная Никласом Луманом. Каждая идея записывается на отдельную "карточку" (файл) и связывается с другими через ссылки. Идеальна для накопления и переплетения знаний.

Пример в Cursor: Папка notes/ с файлами вроде 202603-productivity-rules.md, 202603-morning-routine.md. В каждом файле -- ссылки на связанные заметки.
