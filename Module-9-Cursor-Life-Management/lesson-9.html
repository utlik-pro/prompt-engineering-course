Системы и шаблоны для жизни

GTD, OKR, трекеры привычек, база знаний — всё в Markdown, всё через ИИ

Заметка для преподавателя

Цель урока: Показать студентам, как построить полноценную систему управления жизнью в Cursor, используя проверенные методологии (GTD, OKR, Zettelkasten) и силу ИИ.

Ключевой момент: Это не просто теория — студенты должны выйти с урока с работающей системой, которую они смогут использовать каждый день.

Практика: Каждый студент настраивает свою GTD-систему и трекер привычек. Дайте 20-25 минут на практическую часть.

Зачем нужны системы?

Продуктивный человек отличается от занятого тем, что у него есть система. Система — это набор правил и инструментов, которые автоматизируют принятие решений.

GTD
Getting Things Done

Система управления задачами Дэвида Аллена. Всё из головы — в систему.

OKR
Objectives & Key Results

Система целеполагания из Google. Цели + измеримые результаты.

Zettelkasten
Second Brain

Система заметок с перекрёстными ссылками. Знания, которые растут.

Все эти системы прекрасно реализуются в Cursor через Markdown-файлы + ИИ-ассистент. Никаких платных приложений — только текст и интеллект.

GTD в Cursor: полная реализация

GTD (Getting Things Done) — самая популярная система продуктивности. Её суть: выгрузить всё из головы в надёжную систему и обрабатывать по правилам.

Структура папок GTD:

Файловая структура 
life-system/
  gtd/
    inbox.md              — Входящие: всё, что приходит в голову
    next-actions.md       — Следующие действия: конкретные задачи
    projects.md           — Проекты: задачи из 2+ шагов
    waiting-for.md        — Ожидание: делегировано или жду ответа
    someday-maybe.md      — Когда-нибудь: идеи на будущее
    reference.md          — Справочная: полезная информация
    weekly-review.md      — Еженедельный обзор (шаблон)

Пример: inbox.md

Markdown 
# Inbox — Входящие

Записывайте сюда ВСЁ, что приходит в голову.
Не фильтруйте, не оценивайте — просто запишите.
Обрабатывайте раз в день.

## Новые записи
- Позвонить стоматологу записаться на приём
- Идея: курс по автоматизации для фрилансеров
- Купить подарок маме на день рождения (15 апреля)
- Разобрать фотографии из отпуска
- Ответить на письмо от клиента по проекту X
- Изучить новую функцию Cursor Rules
- Обновить резюме на LinkedIn

Как обрабатывать Inbox через ИИ:

Промпт для Chat 
Обработай мой inbox из @inbox.md по правилам GTD:

1. Для каждого пункта определи:
   - Это действие или справочная информация?
   - Если действие — можно сделать за 2 минуты? (тогда → сделай сейчас)
   - Если нет — это проект (2+ шагов) или одно действие?
   - Какой контекст: @дом, @работа, @компьютер, @телефон?

2. Распредели по файлам:
   - Конкретные действия → next-actions.md
   - Проекты → projects.md
   - Идеи на будущее → someday-maybe.md

3. Для каждого действия укажи контекст и примерное время
Сила ИИ: Обычно обработка inbox занимает 15-20 минут. С ИИ — 2 минуты. Он распределит задачи, добавит контексты и даже предложит приоритеты.
OKR для личных целей

OKR (Objectives and Key Results) — система из Google, которая отлично работает для личных целей. Objective — это вдохновляющая цель, Key Results — измеримые результаты.

okr-q2-2026.md — Пример личных OKR 
# OKR — Q2 2026 (Апрель - Июнь)

## Objective 1: Стать востребованным промпт-инженером
- [ ] KR1: Завершить курс Prompt Engineering 2.0 (прогресс: 9/12 модулей)
- [ ] KR2: Создать портфолио из 5 кейсов использования ИИ
- [ ] KR3: Получить 3 оплаченных заказа на промпт-инжиниринг
- [ ] KR4: Набрать 500 подписчиков в Telegram-канале об ИИ

## Objective 2: Улучшить здоровье и энергию
- [ ] KR1: Тренироваться минимум 3 раза в неделю (12 недель подряд)
- [ ] KR2: Спать 7+ часов каждую ночь (90% дней квартала)
- [ ] KR3: Похудеть на 5 кг (с 85 до 80 кг)
- [ ] KR4: Пить 2 литра воды ежедневно

## Objective 3: Навести порядок в финансах
- [ ] KR1: Накопить 100 000 руб. на подушку безопасности
- [ ] KR2: Сократить расходы на подписки на 30%
- [ ] KR3: Запустить дополнительный источник дохода (фриланс)
- [ ] KR4: Вести ежедневный учёт расходов без пропусков

Трекер привычек в Markdown

Markdown-таблица — идеальный формат для трекера привычек. Простой, наглядный, легко обновлять через ИИ.

habits-march-2026.md 
# Трекер привычек — Март 2026

| Привычка          | Пн | Вт | Ср | Чт | Пт | Сб | Вс | Итого |
|-------------------|----|----|----|----|----|----|----| ------|
| Зарядка 15 мин    | +  | +  | -  | +  | +  | -  | +  | 5/7   |
| Чтение 30 мин     | +  | +  | +  | +  | -  | +  | +  | 6/7   |
| Без соц.сетей до 10:00 | + | - | + | + | + | - | - | 4/7 |
| Медитация 10 мин  | -  | +  | +  | -  | +  | +  | +  | 5/7   |
| Дневник вечером   | +  | +  | +  | +  | +  | -  | +  | 6/7   |
| Вода 2 литра      | +  | +  | +  | -  | +  | +  | +  | 6/7   |
| Прогулка 30 мин   | -  | +  | -  | +  | -  | +  | +  | 4/7   |

## Статистика недели
- Лучшая привычка: Чтение и Дневник (86%)
- Худшая привычка: Без соц.сетей и Прогулка (57%)
- Общий процент выполнения: 73%

Как обновлять через ИИ:

"Добавь новую неделю (10-16 марта) в трекер. Пока все ячейки пустые."

ИИ создаст новую строку дней для заполнения.

"Обнови статистику за неделю. Посчитай процент выполнения каждой привычки и общий процент."

ИИ пересчитает все итоги автоматически.

"Проанализируй мои привычки за месяц. Какие стабильные, какие проседают? Что посоветуешь?"

ИИ найдёт паттерны и даст рекомендации по улучшению.

База знаний: Zettelkasten в Cursor

Zettelkasten — метод заметок, изобретённый немецким социологом Николасом Луманом. Идея: каждая заметка — отдельный файл, связанный с другими через ссылки.

Структура базы знаний 
life-system/
  knowledge/
    inbox/                — Необработанные заметки
      raw-note-001.md
    notes/                — Постоянные заметки (Zettel)
      prompt-engineering-basics.md
      cursor-shortcuts.md
      gtd-principles.md
      ai-for-productivity.md
    maps/                 — Карты содержания (MOC)
      moc-productivity.md
      moc-ai-tools.md
    tags.md               — Индекс тегов

Пример заметки Zettelkasten:

notes/prompt-engineering-basics.md 
# Основы промпт-инженеринга

**Теги:** #prompt-engineering #ai #навыки
**Создано:** 2026-03-10
**Связи:** [[cursor-shortcuts]], [[ai-for-productivity]]

## Суть
Промпт-инженеринг — навык формулирования запросов к ИИ
для получения максимально полезных ответов.

## Ключевые принципы
1. Контекст — дайте ИИ понять ситуацию
2. Конкретность — чем точнее запрос, тем лучше ответ
3. Формат — укажите, в каком виде хотите результат
4. Итерация — уточняйте и улучшайте промпт

## Связанные идеи
- [[cursor-shortcuts]] — Шорткаты ускоряют работу с промптами
- [[ai-for-productivity]] — Промптинг как основа продуктивности с ИИ

## Источники
- Курс Prompt Engineering 2.0, Модуль 9
Как ИИ помогает с базой знаний:
"Создай новую заметку из этого текста по шаблону Zettelkasten. Добавь теги и предложи связи с существующими заметками."
"Проанализируй все заметки в @notes/ и обнови карту содержания @moc-productivity.md."
"Найди связи между @prompt-engineering-basics.md и @gtd-principles.md. Стоит ли создать новую заметку на стыке тем?"

Практика: настраиваем жизненную систему

Выполните три задания. К концу практики у вас будет работающая система продуктивности.

Задание 1: GTD-система (10 мин)

  1. Создайте папку life-system/gtd/
  2. Через Composer создайте все 7 файлов GTD (inbox, next-actions, projects, waiting-for, someday-maybe, reference, weekly-review)
  3. Заполните inbox.md минимум 10 пунктами из головы
  4. Попросите ИИ обработать inbox по правилам GTD и распределить по файлам

Задание 2: Трекер привычек (10 мин)

  1. Создайте файл life-system/habits/march-2026.md
  2. Через Cmd+K или Chat создайте таблицу с 5-7 привычками, которые хотите отслеживать
  3. Заполните данные за текущую неделю
  4. Попросите ИИ посчитать статистику и дать рекомендации

Задание 3: Шаблоны (10 мин)

  1. Создайте папку life-system/templates/
  2. Через Composer создайте три шаблона: утренний ритуал, еженедельный обзор, ретроспектива месяца
  3. Заполните утренний ритуал на сегодня через ИИ
