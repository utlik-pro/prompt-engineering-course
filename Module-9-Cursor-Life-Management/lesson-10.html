Интеграция с внешними сервисами

Подключаем Cursor к внешнему миру: API, сервисы, автоматизации

Заметка для преподавателя

Цель урока: Показать, что Cursor может взаимодействовать с внешними сервисами через API, скрипты и MCP-серверы. Студенты НЕ программисты -- они учатся просить ИИ создавать интеграции.

Важно: Акцент на промпт-инженерии: как правильно описать задачу ИИ, чтобы он сам написал нужный скрипт или запрос. Студенты не пишут код -- они формулируют запросы.

Практика: Покажите живой пример: попросите Cursor через Chat создать скрипт, который делает API-запрос. Результат -- вдохновение, а не зубрёжка синтаксиса.

Cursor + внешний мир

Cursor -- это не замкнутая система. Через терминал и скрипты он может взаимодействовать с любым сервисом в интернете.

Что можно делать

Получать данные из Google Sheets, Notion, Telegram. Запрашивать погоду, курсы валют, новости. Отправлять уведомления.

Как это работает

Вы описываете задачу промптом. ИИ создаёт скрипт. Скрипт запускается в терминале Cursor. Данные возвращаются в ваш проект.

Зачем нужны интеграции?

Интеграции превращают Cursor из текстового редактора в центр управления вашей жизнью:

📊

Данные из таблиц

Подтянуть бюджет из Google Sheets, проанализировать расходы, сгенерировать отчёт

📝

Заметки из Notion

Синхронизировать задачи, перенести базу знаний, обновить статусы проектов

🤖

Боты и уведомления

Telegram-бот для напоминаний, автоматические сводки, оповещения о событиях

Работа с Google Sheets

Google Sheets -- одна из самых полезных интеграций. Вы можете экспортировать и импортировать данные, анализировать таблицы прямо в Cursor.

Вариант 1: Экспорт через CSV

Промпт для Cursor Chat 
У меня есть CSV-файл budget.csv с колонками:
Дата, Категория, Сумма, Описание.

Прочитай этот файл, сгруппируй расходы по категориям,
посчитай итоги и создай Markdown-отчёт с таблицей
и выводами. Сохрани в reports/budget-report.md

Вариант 2: Google Sheets API

Промпт для Cursor Chat 
Создай Python-скрипт, который:
1. Подключается к Google Sheets через API
   (Service Account, credentials.json уже есть)
2. Читает таблицу "Бюджет 2026" (ID: xxx)
3. Скачивает данные за текущий месяц
4. Сохраняет в файл data/budget-march.json

Используй библиотеку gspread.
Простой путь: Не хотите разбираться с API? Просто скачайте таблицу как CSV (Файл -> Скачать -> CSV) и закиньте в папку проекта. Cursor прочитает и проанализирует!
Интеграция с Notion

Notion -- популярная система для заметок и управления проектами. Cursor может забирать из неё данные и обрабатывать.

Перенос задач из Notion

Промпт для Cursor Chat 
Создай скрипт, который через Notion API:
1. Подключается к моей базе данных задач
   (database_id: xxx, token: в .env файле)
2. Получает все задачи со статусом "В работе"
3. Форматирует их в Markdown-список с приоритетами
4. Сохраняет в tasks/notion-tasks.md

Добавь дату синхронизации в начало файла.

Синхронизация заметок

Промпт для Cursor Chat 
У меня есть файл meeting-notes.md с заметками
с совещания. Создай скрипт, который:
1. Прочитает этот файл
2. Создаст новую страницу в Notion
   в разделе "Совещания"
3. Перенесёт содержимое с форматированием
4. Добавит тег "март 2026" и статус "Обработано"

Забрать задачи, заметки, базы данных для анализа в Cursor

Cursor -> Notion

Отправить обработанные отчёты, резюме, планы обратно в Notion

Telegram-бот через Cursor

Создайте бота в Telegram

TELEGRAM_BOT_TOKEN=ваш_токен_от_botfather

3
Попросите Cursor написать бота

Опишите функциональность в промпте

Промпт для Cursor Chat 
Создай простого Telegram-бота на Python (aiogram), который:
1. Принимает команду /remind "текст" "время"
   (например: /remind "Позвонить маме" "18:00")
2. Сохраняет напоминания в JSON-файл
3. Проверяет каждую минуту, не пора ли отправить
4. Отправляет напоминание в нужное время
5. Команда /list -- показывает все напоминания

Токен бота в .env файле (TELEGRAM_BOT_TOKEN).
Сделай код максимально простым, с комментариями.
Результат: Через 5 минут у вас будет работающий Telegram-бот для напоминаний! Запустите его через терминал Cursor: python bot.py
API без программирования

API (Application Programming Interface) -- это способ общения программ друг с другом. Через Cursor вы можете обращаться к любым API, не зная программирования. Просто опишите, что вам нужно.

Примеры полезных API:

Сервис Что можно получить Пример использования
OpenWeatherMap Погода в любом городе Утренняя сводка с погодой на день
ExchangeRate API Курсы валют Ежедневный мониторинг курса доллара
NewsAPI Новости по теме Автоматический дайджест новостей по ИИ
CalDAV Событиия из календаря Список встреч на сегодня
GitHub API Репозитории, issues Статус ваших проектов

Как попросить Cursor сделать API-запрос:

Промпт для Cursor Chat 
Создай скрипт, который:
1. Запрашивает текущую погоду в Минске
   через OpenWeatherMap API (ключ в .env: WEATHER_API_KEY)
2. Запрашивает курс USD/BYN через exchangerate-api.com
3. Форматирует всё в красивую утреннюю сводку
4. Сохраняет в daily/morning-brief-2026-03-12.md

Формат сводки:
# Утренняя сводка -- 12 марта 2026
## Погода
- Температура: ...
- Описание: ...
## Курсы валют
- USD/BYN: ...
Ваша роль: Вы не программируете. Вы описываете, какой результат хотите получить. ИИ сам выберет нужные библиотеки, напишет код и объяснит, как запустить.
MCP-серверы -- расширения Cursor

MCP (Model Context Protocol) -- это способ подключать к Cursor дополнительные инструменты. По сути, это плагины, которые дают ИИ новые возможности.

Без MCP

ИИ работает только с файлами в вашем проекте и может делать запросы через терминал

С MCP

ИИ может напрямую обращаться к базам данных, API, файловым системам, браузеру и другим сервисам

Популярные MCP-серверы:

@web -- поиск информации в интернете прямо из чата
filesystem -- расширенная работа с файловой системой
brave-search -- поиск через Brave для исследований и аналитики
notion -- прямое чтение и запись в Notion без скриптов
github -- управление репозиториями и issues

Как подключить MCP-сервер:

Файл .cursor/mcp.json 
{
  "mcpServers": {
    "brave-search": {
      "command": "npx",
      "args": ["-y", "@anthropic/mcp-brave-search"],
      "env": {
        "BRAVE_API_KEY": "ваш_ключ"
      }
    }
  }
}
Совет: Начните с @web -- он уже встроен в Cursor. Просто добавьте @web в начало промпта, и ИИ будет искать актуальную информацию в интернете.
Пример: Автоматический дайджест

Давайте соберём всё вместе и создадим автоматический новостной дайджест -- скрипт, который собирает информацию из нескольких источников и создаёт отчёт.

1
Сбор новостей

Скрипт запрашивает NewsAPI по вашим темам (ИИ, продуктивность, бизнес)

2
Анализ через ИИ

Cursor Chat анализирует собранные новости, выделяет ключевые тренды

3
Генерация отчёта

ИИ создаёт структурированный Markdown-дайджест с резюме и рекомендациями

Промпт для создания дайджест-системы 
Создай систему автоматического дайджеста:

1. Скрипт fetch-news.py:
   - Запрашивает NewsAPI по темам: AI, productivity, technology
   - Получает 10 статей за последние 24 часа по каждой теме
   - Сохраняет заголовки и описания в data/raw-news.json

2. Файл digest-template.md:
   - Шаблон дайджеста с секциями: Главное, Тренды,
     Рекомендации к прочтению

3. Промпт для анализа (сохрани в prompts/digest-prompt.md):
   "Проанализируй файл data/raw-news.json.
    Выдели 3 главные новости, определи 2 тренда.
    Заполни шаблон digest-template.md.
    Сохрани результат в digests/digest-ДАТА.md"

API-ключ NewsAPI в .env файле (NEWS_API_KEY).
Пример результата: digests/digest-2026-03-12.md 
# Ежедневный дайджест -- 12 марта 2026

## Главные новости
1. **OpenAI представила GPT-5** -- новая модель...
2. **Apple интегрирует ИИ в iWork** -- Pages и Numbers...
3. **Рынок no-code вырос на 40%** -- аналитики прогнозируют...

## Тренды недели
- ИИ-ассистенты становятся персональными
- Рост спроса на промпт-инженеров в СНГ

Архитектура интеграций

Вот как выглядит типичная схема интеграций в вашем проекте:

Структура папок 
my-life-system/
├── .env                    # API-ключи (НИКОГДА не делитесь!)
├── .cursor/
│   └── mcp.json           # Настройки MCP-серверов
├── scripts/
│   ├── fetch-news.py      # Скрипт сбора новостей
│   ├── fetch-weather.py   # Скрипт погоды
│   ├── sync-notion.py     # Синхронизация с Notion
│   └── telegram-bot.py    # Telegram-бот
├── data/
│   ├── raw-news.json      # Сырые данные
│   └── budget.csv         # Данные из Google Sheets
├── digests/
│   └── digest-2026-03-12.md
├── prompts/
│   └── digest-prompt.md   # Промпт для анализа
└── templates/
    └── digest-template.md # Шаблон дайджеста
Безопасность: Всегда храните API-ключи в файле .env и добавьте его в .gitignore. Никогда не вставляйте ключи прямо в код или промпты.

Практическое задание

Выполните два задания:

Задание 1: Подключите API

  1. Зарегистрируйтесь на openweathermap.org (бесплатный аккаунт)
  2. Получите API-ключ и сохраните в .env
  3. Попросите Cursor создать скрипт для получения погоды
  4. Запустите скрипт в терминале и проверьте результат

Задание 2: Создайте дайджест

  1. Создайте шаблон дайджеста в templates/digest-template.md
  2. Найдите 5-10 интересных статей вручную (или через @web)
  3. Сохраните ссылки и описания в файл data/raw-articles.md
  4. Попросите Cursor проанализировать статьи и заполнить шаблон дайджеста
  5. Сохраните итоговый дайджест с сегодняшней датой
Бонус: Попробуйте настроить MCP-сервер @web и создать дайджест целиком через Chat, без ручного поиска статей!
