Управление личными проектами

Превращаем любую жизненную цель в управляемый проект с помощью Cursor

Проект = папка в Cursor

Ключевая идея этого урока: любая жизненная цель может быть организована как проект в Cursor. Переезд в другой город, запуск бизнеса, подготовка к свадьбе, получение нового образования — всё это проекты, которые состоят из задач, сроков и ресурсов.

📁
Традиционный подход

Заметки в телефоне, стикеры, Excel-таблицы, записи в блокноте — хаос и потерянная информация

⚡
Подход Cursor

Структурированная папка с файлами, ИИ-помощник всегда рядом, всё в одном месте

Когда вы открываете папку в Cursor — ИИ видит все файлы внутри неё. Это значит, что он понимает контекст вашего проекта целиком: задачи, заметки, сроки, бюджет. Вы можете задавать вопросы типа "Что мне делать дальше?" или "Какие задачи просрочены?" — и получать осмысленные ответы.

Какие жизненные проекты можно вести в Cursor

Cursor подходит для управления абсолютно любым проектом. Вот примеры из разных сфер жизни:

Переезд в другой город

Поиск жилья, оформление документов, логистика, бюджет, контакты риелторов

Запуск онлайн-бизнеса

Бизнес-план, регистрация, маркетинг, создание продукта, финансы

Подготовка к свадьбе

Список гостей, площадка, меню, декор, бюджет, таймлайн, подрядчики

Получение образования

Выбор программы, документы, подготовка, расписание, конспекты

Ремонт квартиры

Дизайн-проект, смета, подрядчики, закупки, сроки, контроль

Организация мероприятия

Программа, спикеры, логистика, продвижение, билеты, спонсоры

Структура папок проекта

Для любого проекта используйте единый шаблон структуры. Это позволит ИИ лучше понимать контекст и давать более точные ответы.

Универсальная структура проекта 
my-project/
├── README.md          # Описание проекта: цель, сроки, ключевые решения
├── tasks.md           # Список задач со статусами
├── timeline.md        # Сроки и дедлайны
├── budget.md          # Бюджет (если применимо)
├── notes/             # Заметки и размышления
│   ├── ideas.md       # Идеи и мозговой штурм
│   ├── meetings.md    # Заметки со встреч
│   └── decisions.md   # Принятые решения и их обоснования
├── resources/         # Полезные ресурсы
│   ├── contacts.md    # Контакты людей
│   ├── links.md       # Полезные ссылки
│   └── templates/     # Шаблоны документов
└── archive/           # Завершённые или неактуальные материалы

Что включить в каждый файл:

README.md — главный файл проекта

Цель проекта, почему это важно, ключевые даты, критерии успеха, текущий статус. ИИ будет обращаться к этому файлу для понимания контекста.

tasks.md — задачи и их статусы

Каждая задача с чекбоксом, приоритетом и дедлайном. Можно группировать по категориям или этапам.

timeline.md — календарный план

Этапы проекта с датами начала и окончания. Вехи (milestones) и ключевые дедлайны.

notes/ — папка с заметками

Свободное пространство для мыслей, идей, заметок со встреч. ИИ видит всё и может анализировать.

Пример README.md 
# Запуск онлайн-курса по фотографии

## Цель
Создать и запустить онлайн-курс по мобильной фотографии
для начинающих. Первый поток — 30 учеников.

## Сроки
- Старт подготовки: 1 марта 2026
- Запуск продаж: 1 мая 2026
- Старт курса: 1 июня 2026

## Критерии успеха
- Минимум 20 учеников на первом потоке
- Средняя оценка курса не ниже 4.5/5
- Окупаемость затрат в первый месяц

## Текущий статус
Этап: подготовка контента (40% готово)

## Ключевые решения
- Платформа: Stepik
- Формат: видеоуроки + практические задания
- Цена: $49 за курс
Создание проекта: пошаговый процесс

Вот точная последовательность действий для запуска нового проекта в Cursor:

1
Создайте папку на компьютере

Создайте папку с понятным названием, например: wedding-planning, new-business, apartment-renovation

2
Откройте папку в Cursor

File → Open Folder → выберите созданную папку. Теперь это ваше рабочее пространство.

3
Попросите ИИ создать структуру

Откройте Chat и опишите свой проект. ИИ предложит оптимальную структуру файлов.

4
Заполните README.md

Опишите цель, сроки и критерии успеха. Это фундамент вашего проекта.

5
Декомпозируйте цель на задачи

Попросите ИИ разбить вашу цель на конкретные шаги и задачи. Заполните tasks.md.

Промпт для создания структуры 
Я планирую [описание проекта].

Создай для меня структуру проекта:
1. README.md с описанием цели, сроками и критериями успеха
2. tasks.md со списком всех задач, разбитых по этапам
3. timeline.md с календарным планом
4. budget.md с примерной сметой

Контекст: [дополнительная информация о проекте]
Сроки: [когда нужно завершить]
Бюджет: [примерный бюджет]
Декомпозиция целей с ИИ

Декомпозиция — это разбиение большой цели на маленькие управляемые шаги. Это самый важный навык управления проектами, и ИИ делает его невероятно простым.

Принцип трёх уровней:

Уровень 1: Цель

Запустить онлайн-курс по фотографии

Уровень 2: Этапы

Исследование рынка → Создание программы → Запись контента → Настройка платформы → Маркетинг → Запуск

Уровень 3: Задачи

Для этапа "Исследование рынка": изучить конкурентов, провести опрос ЦА, определить нишу, сформулировать УТП...

Промпт для декомпозиции 
Моя цель: запустить онлайн-курс по мобильной фотографии
для начинающих за 3 месяца.

Разбей эту цель на:
1. Основные этапы (5-7 этапов)
2. Для каждого этапа — конкретные задачи (3-5 задач)
3. Для каждой задачи укажи:
   - Примерное время выполнения
   - Зависимости от других задач
   - Приоритет (высокий/средний/низкий)

Оформи в виде Markdown с чекбоксами [ ]
Результат: tasks.md 
# Задачи: Онлайн-курс по фотографии

## Этап 1: Исследование (неделя 1-2)
- [ ] Изучить 10 конкурентных курсов (2 дня) [высокий]
- [ ] Провести опрос в соцсетях — 50 ответов (3 дня) [высокий]
- [ ] Определить нишу и целевую аудиторию (1 день) [высокий]
- [ ] Сформулировать УТП курса (1 день) [средний]

## Этап 2: Программа курса (неделя 3-4)
- [ ] Составить структуру курса: модули и уроки (2 дня) [высокий]
- [ ] Написать описание каждого урока (3 дня) [высокий]
- [ ] Подготовить практические задания (3 дня) [средний]
- [ ] Согласовать программу с экспертом (2 дня) [низкий]

## Этап 3: Создание контента (неделя 5-8)
- [ ] Написать скрипты для видеоуроков (5 дней) [высокий]
- [ ] Записать видеоуроки (7 дней) [высокий]
- [ ] Смонтировать видео (5 дней) [высокий]
- [ ] Подготовить дополнительные материалы (3 дня) [средний]

## Этап 4: Платформа (неделя 9-10)
- [ ] Выбрать и настроить платформу (2 дня) [высокий]
- [ ] Загрузить весь контент (2 дня) [высокий]
- [ ] Настроить автоматизации (1 день) [средний]
- [ ] Тестовый прогон с 3 друзьями (3 дня) [высокий]

## Этап 5: Маркетинг и запуск (неделя 11-12)
- [ ] Создать лендинг курса (2 дня) [высокий]
- [ ] Подготовить email-воронку (2 дня) [средний]
- [ ] Запустить рекламу в соцсетях (1 день) [высокий]
- [ ] Провести бесплатный вебинар (1 день) [высокий]
- [ ] Открыть продажи (1 день) [высокий]
Трекинг прогресса

Markdown-чеклисты — простой и мощный инструмент для отслеживания прогресса. А ИИ поможет вам поддерживать их в актуальном состоянии.

Как работают чеклисты в Markdown:

Синтаксис чеклистов 
- [ ] Незавершённая задача
- [x] Завершённая задача
- [ ] Ещё одна задача (дедлайн: 15 марта)

Обновление прогресса через ИИ:

Вместо того чтобы вручную искать задачи и менять их статус, просто скажите ИИ:

Промпт 
Отметь следующие задачи как выполненные в tasks.md:
- Изучить конкурентов
- Провести опрос
И добавь новую задачу в этап "Исследование":
- Проанализировать результаты опроса (1 день) [высокий]
Промпт для обзора прогресса 
Посмотри на мой tasks.md и скажи:
1. Сколько процентов задач завершено?
2. Какие задачи с высоким приоритетом ещё не сделаны?
3. Укладываюсь ли я в сроки из timeline.md?
4. Что мне стоит сделать на этой неделе?
Совет: Заведите привычку каждое утро открывать Cursor и спрашивать: "Какие задачи мне нужно сделать сегодня?" ИИ проанализирует все ваши файлы и даст приоритизированный список.
Кейс: Запуск онлайн-курса

Разберём полный пример от идеи до реализации. Допустим, вы хотите создать и запустить онлайн-курс.

Шаг 1: Инициализация проекта

Промпт в Chat 
Я хочу создать онлайн-курс по мобильной фотографии.
Целевая аудитория: начинающие фотографы 25-40 лет.
Бюджет: $500. Срок подготовки: 3 месяца.

Создай полную структуру проекта с файлами:
README.md, tasks.md, timeline.md, budget.md,
notes/ideas.md, resources/contacts.md

Шаг 2: Глубокая проработка каждого этапа

После создания структуры, углубляйтесь в каждый этап:

Промпт 
Посмотри на tasks.md, этап "Создание контента".
Для задачи "Написать скрипты видеоуроков" — помоги мне:
1. Определить оптимальную длину каждого урока
2. Составить структуру скрипта (шаблон)
3. Написать скрипт для первого урока "Основы композиции"

Сохрани результат в notes/script-template.md

Шаг 4: Еженедельный обзор

Промпт для еженедельного обзора 
Проведи еженедельный обзор моего проекта:
1. Проверь tasks.md — что сделано за эту неделю?
2. Проверь timeline.md — идём ли мы по графику?
3. Проверь budget.md — нет ли перерасхода?
4. Что запланировать на следующую неделю?

Обнови tasks.md и timeline.md при необходимости.
Кейс: Планирование путешествия

Ещё один частый проект — планирование путешествия. Cursor превратит подготовку поездки в организованный процесс.

Промпт для создания проекта 
Помоги спланировать путешествие в Грузию на 10 дней.
Двое взрослых, бюджет $2000.
Интересы: горы, еда, вино, история.
Даты: 15-25 июля 2026.

Создай проект со следующими файлами:
1. README.md — обзор поездки
2. itinerary.md — маршрут по дням
3. budget.md — бюджет по категориям
4. packing-list.md — список вещей
5. bookings.md — что нужно забронировать
6. food-spots.md — рестораны и кафе для посещения

Еженедельные обзоры

Каждую пятницу просите ИИ проанализировать весь проект и создать отчёт о прогрессе. Сохраняйте отчёты в notes/weekly/.

Файл решений

Ведите notes/decisions.md, где фиксируйте каждое важное решение с обоснованием. Это поможет не забыть, почему вы выбрали именно этот путь.

Управление рисками

Попросите ИИ проанализировать проект на возможные риски и создать файл risks.md с планами их минимизации.

📄
Шаблоны для новых проектов

Создайте папку-шаблон project-template/ с пустыми файлами. При начале нового проекта просто копируйте её.

Промпт для анализа рисков 
Проанализируй мой проект (README.md, tasks.md, timeline.md)
и определи:
1. Топ-5 рисков, которые могут помешать завершению в срок
2. Для каждого риска:
   - Вероятность (высокая/средняя/низкая)
   - Влияние на проект
   - План минимизации
3. Общая оценка: успею ли я завершить проект в срок?

Сохрани в risks.md
