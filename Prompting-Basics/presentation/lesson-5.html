Few-shot Learning: Учим на примерах

Цели урока:

  • Понять разницу между Zero-shot, One-shot и Few-shot
  • Изучить формат подачи примеров
  • Научиться выбирать правильные примеры
  • Применить Few-shot для классификации и форматирования
Заметка для преподавателя

Время: 15 минут

Подход: Покажите пример классификации отзывов с и без примеров. Разница будет очевидна.

Практика: Пусть студенты создадут свой few-shot промпт для задачи из их сферы.

Концепция: обучение на примерах

Few-shot Learning — это техника, при которой вы показываете AI несколько примеров желаемого результата, и он понимает паттерн.

Аналогия: Вместо того чтобы объяснять ребёнку правила грамматики, вы показываете ему правильные предложения. Он сам выводит паттерн.

Три режима промптинга

Zero-shot (0 примеров)

Просто инструкция, без примеров.

Классифицируй этот отзыв как позитивный или негативный:
"Отличный продукт, рекомендую!"

One-shot (1 пример)

Один пример для понимания формата.

Классифицируй отзыв:

Пример:
Отзыв: "Товар пришёл сломанным" → Негативный

Теперь классифицируй:
Отзыв: "Отличный продукт, рекомендую!"

Few-shot (2-5 примеров)

Несколько примеров для надёжного понимания.

Классифицируй отзывы:

Примеры:
"Товар пришёл сломанным" → Негативный
"Всё супер, спасибо!" → Позитивный
"Нормально, ничего особенного" → Нейтральный

Теперь классифицируй:
"Отличный продукт, рекомендую!"

Формат примеров: Input → Output

Примеры должны быть структурированы чётко:

Базовый формат 
Input: [входные данные]
Output: [желаемый результат]

Input: [входные данные]
Output: [желаемый результат]

Input: [новые данные для обработки]
Output:
Важно: Оставляйте "Output:" пустым для финального запроса — AI заполнит его сам.

Как выбирать примеры?

  • Разнообразие: покажите разные случаи (позитивный, негативный, нейтральный)
  • Репрезентативность: примеры должны быть похожи на реальные данные
  • Краткость: не перегружайте, 2-5 примеров достаточно
  • Консистентность: используйте единый формат во всех примерах

Когда что использовать?

Режим Когда использовать
Zero-shot Простые, стандартные задачи. AI уже знает формат.
One-shot Нужно показать формат, но задача типовая.
Few-shot Уникальный формат, классификация, стиль, сложные паттерны.

Комбинирование техник

Few-shot + CoT = Мощная комбинация

Вы можете показать не только результат, но и процесс рассуждения:

Определи тональность и объясни почему:

Отзыв: "Доставка быстрая, но товар оказался бракованным"
Рассуждение: Есть позитивный момент (быстрая доставка), но негативный (брак) перевешивает, так как это основная проблема.
Тональность: Негативная

Отзыв: "Цена высокая, но качество того стоит!"
Рассуждение: Упоминается недостаток (цена), но он оправдывается качеством. Общее впечатление положительное.
Тональность: Позитивная

Отзыв: "Заказ пришёл вовремя, упаковка помялась, но сам товар целый"
Рассуждение:

Проверка понимания

Сколько примеров оптимально использовать в Few-shot промпте?
Чем больше, тем лучше (10-20 примеров)
2-5 разнообразных примеров
Всегда ровно 3 примера
Примеры не нужны, достаточно инструкции

Ключевые выводы

  • Few-shot Learning — обучение AI на примерах
  • Zero/One/Few: 0, 1 или 2-5 примеров соответственно
  • Формат: Input → Output, чёткий и консистентный
  • Выбор примеров: разнообразные, репрезентативные, краткие
  • Применение: классификация, извлечение данных, форматирование, стиль
  • Комбинируйте: Few-shot + CoT для сложных задач
