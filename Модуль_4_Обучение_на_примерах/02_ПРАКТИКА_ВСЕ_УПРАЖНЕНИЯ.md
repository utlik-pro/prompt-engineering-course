# Практический блок: 8 упражнений (90 минут)

## 🎯 Структура практики

- **Упражнение 1-2:** Базовые Few-shot промпты (20 мин)
- **Упражнение 3-4:** Классификация и форматирование (25 мин)
- **Упражнение 5-6:** Оптимизация количества примеров (25 мин)
- **Упражнение 7-8:** Универсальные промпты для реальных задач (20 мин)

**Формат:** Каждое упражнение = демонстрация (5 мин) + самостоятельная работа (10 мин) + обсуждение (5 мин)

---

## 📝 УПРАЖНЕНИЕ 1: Первый Few-shot промпт (10 минут)

### 🎯 Цель

**ДЕМОНСТРАЦИЯ (4 мин):**

```

**Результат:** Скучный, формальный заголовок

**ШАГ 2: Few-shot промпт с примерами**
```

```

**Результат:** Намного лучше!

**ЧТО ГОВОРИТЬ:**

**ДЕМОНСТРАЦИЯ (6 мин):**

**Открыть Claude или ChatGPT и показать:**

## 📝 УПРАЖНЕНИЕ 5: Оптимизация количества примеров (15 минут)

Научиться находить оптимальное количество примеров. Важно: модели 2026 года (GPT-5, Claude 4.6, Grok 4) требуют МЕНЬШЕ примеров, чем модели прошлого поколения. 3-5 примеров часто достаточно там, где раньше нужно было 10-20.

### 🛠 Инструмент
**Grok 4** или **ChatGPT (GPT-5)**

### 📋 Что говорить преподавателю

**ВВЕДЕНИЕ (2 мин):**

> "Больше примеров ≠ лучше результат. Слишком много примеров тратят контекст и деньги (токены). Слишком мало — модель не понимает паттерн. Найдем баланс."

**ЭКСПЕРИМЕНТ:** Классификация тона сообщения

**Вариант 1: 1 пример (недостаточно)**
```
Определи тон сообщения: дружелюбный, нейтральный или агрессивный.

Пример:
"Привет! Как дела? Давно не виделись!" → дружелюбный

Определи тон:
"Вы не ответили на мой вопрос." → ?
```

**Результат:** Скорее всего неточный

**Вариант 2: 3 примера (обычно достаточно)**
```
Определи тон сообщения: дружелюбный, нейтральный или агрессивный.

Примеры:
"Привет! Как дела? Давно не виделись!" → дружелюбный
"Отправил документы. Проверьте." → нейтральный
"Вы опять не сделали работу! Это неприемлемо!" → агрессивный

Определи тон:
"Вы не ответили на мой вопрос." → ?
```

**Результат:** Точный (нейтральный)

**Вариант 3: 10 примеров (избыточно)**
```
[10 примеров тонов]

Определи тон:
"Вы не ответили на мой вопрос." → ?
```

**Результат:** Такой же, как с 3 примерами, но потрачено больше токенов

**ВЫВОД:**

> "Правило большого пальца:
> - Простая задача (2-3 категории): 3-5 примеров
> - Средняя задача (4-6 категорий): 6-10 примеров
> - Сложная задача (7+ категорий или edge cases): 10-20 примеров
>
> Начинайте с 3, увеличивайте только если нужно."

### 👨‍💻 ЗАДАНИЕ ДЛЯ СТУДЕНТОВ (10 мин)

**ИНСТРУКЦИЯ:**

> "Проведите эксперимент: создайте классификатор с разным количеством примеров и сравните результаты.
>
> **Задача:** Классификация вопросов на форуме (Техническая проблема / Вопрос новичка / Обсуждение / Оффтоп)
>
> **Шаги:**
> 1. Создайте промпт с 1 примером на категорию (всего 4)
> 2. Протестируйте на 5 вопросах, запишите точность
> 3. Создайте промпт с 2 примерами на категорию (всего 8)
> 4. Протестируйте на тех же 5 вопросах, запишите точность
> 5. Создайте промпт с 3 примерами на категорию (всего 12)
> 6. Протестируйте, запишите точность
>
> **Заполните таблицу:**
> | Кол-во примеров | Точность | Токены* |
> |-----------------|----------|---------|
> | 4               | X/5      | ~       |
> | 8               | X/5      | ~       |
> | 12              | X/5      | ~       |
>
> *Примерно оцените длину промпта"

**ТЕСТОВЫЕ ВОПРОСЫ:**
```
1. "У меня вылетает приложение при открытии файлов больше 100MB"
2. "Как начать использовать эту программу? Где скачать?"
3. "Как вы думаете, какой язык программирования лучше учить в 2026?"
4. "Кто смотрел новый сезон сериала?"
5. "Ошибка 'Cannot read property of undefined', как исправить?"
```

### 💬 ОБСУЖДЕНИЕ (8 мин)

**СОБРАТЬ РЕЗУЛЬТАТЫ НА ДОСКЕ:**

```
Студент 1:
4 примера → 2/5 (40%)
8 примеров → 4/5 (80%)
12 примеров → 5/5 (100%)

Студент 2:
4 примера → 3/5 (60%)
8 примеров → 5/5 (100%)
12 примеров → 5/5 (100%)
```

**ВОПРОСЫ:**
1. "На каком количестве примеров точность перестала расти?"
2. "Стоило ли добавлять 12 примеров вместо 8?"
3. "Какие ошибки делала модель с малым количеством примеров?"

**КЛЮЧЕВОЙ ВЫВОД:**

> "Обычно есть 'точка насыщения' — после неё больше примеров не улучшают результат. Для большинства задач это 6-10 примеров. Не тратьте токены зря — найдите эту точку."

**ПРАВИЛО ОПТИМИЗАЦИИ:**
```
1. Начните с 3 примеров
2. Протестируйте на 10+ примерах
3. Если точность < 80% → добавьте примеры
4. Если точность > 90% → уберите лишние примеры
5. Найдите минимум, при котором точность > 90%
```

## 📝 УПРАЖНЕНИЕ 6: Мультимодальный Few-shot + улучшение примеров (10 минут)

### 🎯 Цель
Научиться улучшать примеры для повышения точности. НОВОЕ: мультимодальный few-shot с изображениями.

### 🛠 Инструмент
**Gemini 3.1 Pro** (лучший для мультимодального few-shot — до 900 изображений) или **Claude Sonnet 4.6**

### 📋 Что говорить преподавателю

**ВВЕДЕНИЕ (2 мин):**

> "Дело не только в количестве, но и в качестве примеров. Плохие примеры → плохой результат. А в 2026 году у нас есть мощная новая возможность — мультимодальный few-shot: показываем модели не только текстовые примеры, но и изображения!"

**МУЛЬТИМОДАЛЬНЫЙ FEW-SHOT (НОВОЕ):**

> "В Gemini 3.1 Pro вы можете прикрепить до 900 изображений в одном промпте. Это значит, что можно учить модель классифицировать скриншоты UI, анализировать фотографии товаров, распознавать дефекты на производстве — всё через few-shot примеры с картинками."

**Пример мультимодального few-shot:**
```
Определи тип дефекта на фотографии товара.

Пример 1: [фото царапины] → "Царапина на поверхности"
Пример 2: [фото вмятины] → "Вмятина на корпусе"
Пример 3: [фото без дефектов] → "Дефектов не обнаружено"

Определи дефект: [новое фото]
```

> "Также Claude и GPT-5 поддерживают мультимодальный few-shot. Gemini 3.1 Pro дополнительно принимает аудио и видео (YouTube URL прямо в промпте!)."

**ДЕМОНСТРАЦИЯ УЛУЧШЕНИЯ ТЕКСТОВЫХ ПРИМЕРОВ (3 мин):**

**ПЛОХИЕ ПРИМЕРЫ (все слишком простые):**
```
Задача: Извлечь ключевые слова из описания вакансии

Пример 1:
"Python developer" → python, developer

Пример 2:
"Java engineer" → java, engineer

Пример 3:
"React frontend" → react, frontend

---

Извлеки ключевые слова:
"Senior Full-Stack Developer (Python/React) with 5+ years experience in microservices architecture"
```

**Результат:** Скорее всего неполный (не извлекает "senior", "full-stack", "5+ years", "microservices")

**ХОРОШИЕ ПРИМЕРЫ (разнообразные, с edge cases):**
```
Задача: Извлечь ключевые слова из описания вакансии

Пример 1:
"Senior Python Developer with Django" → senior, python, developer, django

Пример 2:
"Junior Java Engineer (remote, up to $80k)" → junior, java, engineer, remote, up to $80k

Пример 3:
"Full-Stack Developer: React + Node.js + MongoDB, 3+ years experience" → full-stack, developer, react, node.js, mongodb, 3+ years experience

---

Извлеки ключевые слова:
"Senior Full-Stack Developer (Python/React) with 5+ years experience in microservices architecture"
```

**Результат:** Намного полнее!

**ЧТО ГОВОРИТЬ:**

> "Что изменилось:
> 1. Разные уровни (senior, junior, middle)
> 2. Разные форматы (с скобками, с зарплатой)
> 3. Разная длина (короткие и длинные)
> 4. Edge cases (составные термины 'full-stack', диапазоны '3+ years')"
