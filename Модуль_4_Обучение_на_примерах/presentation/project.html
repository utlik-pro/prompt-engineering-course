Домашнее задание: Few-shot Prompting

Применение полученных знаний на практике. Пять задач разного уровня сложности для закрепления навыков Few-shot промптинга.

Цель задания

Создать практические Few-shot промпты для реальных бизнес-задач и понять разницу между различными подходами к промптингу.

Задание 1: Корпоративный стиль (20 баллов)

Создать Few-shot промпт для генерации описаний в стиле вашей компании

Задача: Разработать промпт с 3-5 примерами, который научит AI писать в фирменном стиле вашей компании (или выдуманной компании).

Что нужно сделать:

  1. Выберите компанию (реальную или придумайте свою)
  2. Определите характеристики стиля:
    • Тон (формальный/дружелюбный/игривый/профессиональный)
    • Лексика (простая/техническая/образная)
    • Длина предложений (короткие/средние/длинные)
    • Использование эмодзи, восклицательных знаков и т.д.
  3. Создайте 3-5 примеров описаний продуктов/услуг в этом стиле
  4. Протестируйте промпт на новом продукте
Пример структуры
Шаблон:
Задача: Написать описание [продукта/услуги] в стиле компании [название].

Стиль компании: [краткое описание tone of voice]

Пример 1:
Продукт: [название]
Описание: [текст в нужном стиле]

Пример 2:
...

---

Теперь напиши описание для:
Продукт: [новый продукт]

Критерии оценки:

  • Четко определенный стиль (5 баллов)
  • Качество и разнообразие примеров (10 баллов)
  • Результат соответствует заданному стилю (5 баллов)
Задание 2: Email-классификатор (20 баллов)

Построить классификатор email по 4 категориям с 3 примерами на каждую

Задача: Создать промпт для автоматической сортировки входящих писем по приоритетам и категориям.

Категории для классификации:

  1. Срочные — требуют немедленного ответа
  2. Важные — нужен ответ в течение дня
  3. Обычные — можно обработать позже
  4. Информационные — не требуют ответа

Что нужно сделать:

  1. Создать по 3 примера email для каждой категории (всего 12 примеров)
  2. Каждый пример должен включать:
    • Тему письма
    • Краткое содержание или первые строки
    • Категорию
  3. Протестировать на 3-5 новых письмах
  4. Убедиться, что классификация работает корректно
Пример структуры
Шаблон:
Классифицируй email по категориям: Срочные, Важные, Обычные, Информационные.

Тема: "URGENT: Production down"
Текст: "Our main server is not responding..."
Категория: Срочные

Тема: "Встреча завтра в 10:00"
Текст: "Напоминаю о встрече с клиентом..."
Категория: Важные

[еще 10 примеров...]

---

Классифицируй:
Тема: [тема письма]
Текст: [начало письма]
Категория: ?

Критерии оценки:

  • Полнота примеров (3 на категорию) (8 баллов)
  • Качество и реалистичность примеров (8 баллов)
  • Точность классификации на тестовых данных (4 балла)
Задание 3: Извлечение данных в JSON (25 баллов)

Разработать промпт для извлечения структурированных данных из текста

Задача: Создать Few-shot промпт, который извлекает информацию из неструктурированного текста и возвращает её в формате JSON.

Сценарий: Извлечение информации о вакансиях из описаний

Целевая JSON-структура:

{
  "position": "название должности",
  "company": "название компании",
  "location": "город/удаленка",
  "salary": {
    "min": число,
    "max": число,
    "currency": "валюта"
  },
  "requirements": ["требование1", "требование2"],
  "experience_years": число
}

Что нужно сделать:

  1. Создать 4-5 примеров:
    • Входной текст (описание вакансии)
    • Выходной JSON (извлеченные данные)
  2. Примеры должны охватывать разные случаи:
    • С указанием зарплаты и без
    • Удаленная работа и офис
    • Разное количество требований
    • Разные форматы записи опыта
  3. Протестировать на 2-3 новых описаниях
  4. Проверить валидность получаемого JSON
Пример
Входной текст:
Ищем Senior Python разработчика в Минск. Зарплата от 3000$ до 5000$.
Требования: Python, Django, PostgreSQL, 5+ лет опыта.
Выходной JSON:
{
  "position": "Senior Python Developer",
  "company": null,
  "location": "Минск",
  "salary": {
    "min": 3000,
    "max": 5000,
    "currency": "USD"
  },
  "requirements": ["Python", "Django", "PostgreSQL"],
  "experience_years": 5
}

Критерии оценки:

  • Корректность JSON-структуры (8 баллов)
  • Разнообразие edge cases в примерах (10 баллов)
  • Точность извлечения на тестовых данных (7 баллов)
Задание 4: Сравнение Zero-shot vs Few-shot (20 баллов)

Сравнить Zero-shot и Few-shot на одной задаче, описать разницу

Задача: Провести эксперимент и задокументировать различия в результатах.

Что нужно сделать:

  1. Выберите одну из задач:
    • Генерация заголовков для статей
    • Классификация тональности отзывов (позитив/негатив/нейтрал)
    • Создание кратких резюме текстов
    • Перевод технических терминов
  2. Создайте Zero-shot промпт
  3. Протестируйте на 5 примерах, запишите результаты
  4. Создайте Few-shot промпт с 3-4 примерами
  5. Протестируйте на тех же 5 примерах
  6. Напишите сравнительный анализ

Структура анализа:

Шаблон для отчета:
  • 1. Выбранная задача: [описание]
  • 2. Zero-shot промпт: [текст промпта]
  • 3. Few-shot промпт: [текст промпта с примерами]
  • 4. Результаты тестирования:
    • Тест 1: Zero-shot [...] vs Few-shot [...]
    • Тест 2: ...
    • ...
  • 5. Анализ различий:
    • Качество результатов
    • Консистентность
    • Соответствие ожиданиям
  • 6. Выводы: Когда какой подход лучше использовать

Критерии оценки:

  • Корректность обоих промптов (6 баллов)
  • Качество тестирования (5 баллов)
  • Глубина анализа различий (6 баллов)
  • Практические выводы (3 балла)
Задание 5: Универсальный шаблон (15 баллов)

Создать универсальный Few-shot шаблон для повторяющейся задачи

Задача: Разработать переиспользуемый шаблон промпта для регулярной рабочей задачи.

Что нужно сделать:

  1. Выберите задачу, которую выполняете регулярно (или придумайте):
    • Написание отчетов
    • Ответы на типовые вопросы клиентов
    • Создание постов для соцсетей
    • Форматирование документов
    • Генерация тестовых данных
  2. Создайте шаблон с:
    • Местами для подстановки переменных (отмечены [PLACEHOLDER])
    • 3-4 базовыми примерами
    • Четкими инструкциями
    • Описанием формата вывода
  3. Напишите инструкцию по использованию шаблона
  4. Покажите 2-3 примера применения шаблона
Пример структуры
Шаблон:
/* === ШАБЛОН: [НАЗВАНИЕ ЗАДАЧИ] === */

Советы по выполнению
  • Начните с простого: Сначала сделайте Zero-shot версию, потом улучшайте примерами
  • Разнообразие примеров: Каждый пример должен показывать разные аспекты задачи
  • Тестируйте: Пробуйте промпт на edge cases — необычных, граничных ситуациях
