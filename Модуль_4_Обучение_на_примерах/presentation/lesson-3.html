Prompt Ensembling: Объединение промптов

Prompt Ensembling — техника комбинирования нескольких вариаций промптов для получения более надёжных и качественных результатов. Аналогия с комитетом экспертов: несколько "голосов" дают более взвешенное решение.

Основная идея

Вместо одного промпта запускаем несколько вариаций и агрегируем результаты. Это снижает случайные ошибки и повышает стабильность.

Зачем нужен Ensembling?

🎲

Проблема: Нестабильность

Один и тот же промпт может давать разные результаты при повторных запусках. Особенно при temperature > 0.

🎯

Решение: Агрегация

Несколько запусков + голосование/усреднение дают стабильный и более точный результат.

🧠

Принцип: Мудрость толпы

Множество независимых оценок в среднем точнее одной экспертной — принцип из теории решений.

Типы Prompt Ensembling

Тип 1

Один промпт — несколько запусков

Запускаем один и тот же промпт N раз и агрегируем результаты.

Пример: Классификация тональности
Один промпт, 5 запусков:
Определи тональность отзыва: Позитивная, Негативная, Нейтральная.

Отзыв: "Товар неплохой, но доставка подкачала"

Запуск 1: Нейтральная
Запуск 2: Негативная
Запуск 3: Нейтральная
Запуск 4: Нейтральная
Запуск 5: Негативная

Голосование: 3 из 5 = Нейтральная ✓
Когда использовать: Для задач классификации, когда нужна высокая точность и есть бюджет на множественные запросы.
Тип 2

Разные формулировки — один запуск каждой

Создаём несколько разных промптов для одной задачи и сравниваем результаты.

Пример: Оценка качества кода
Промпт A (Прямой вопрос):
Оцени качество этого кода от 1 до 10.
Результат: 7
Промпт B (С критериями):
Оцени код по критериям: читаемость, эффективность, безопасность. Дай общую оценку 1-10.
Результат: 6
Промпт C (Ролевой):
Ты senior разработчик на code review. Оцени код от 1 до 10.
Результат: 6
Агрегация:
Среднее: (7 + 6 + 6) / 3 = 6.33 ≈ 6
Преимущество: Разные формулировки "смотрят" на задачу с разных сторон, снижая bias одного подхода.
Тип 3

Разные роли — совещание экспертов

Просим AI "быть" разными экспертами и собираем их мнения.

Пример: Оценка бизнес-идеи
Маркетолог:
"Как маркетолог, оцени рыночный потенциал идеи"
→ "Рынок насыщен, но есть ниша для B2B сегмента"
Финансист:
"Как финансовый аналитик, оцени окупаемость"
→ "ROI положительный через 18-24 месяца"
Технический директор:
"Как CTO, оцени техническую реализуемость"
→ "Реализуемо, но потребует 6 месяцев разработки"
Итоговая оценка:
Идея жизнеспособна при условии:
- Фокус на B2B сегмент
- Бюджет на 18+ месяцев
- Команда разработки на 6 месяцев MVP

Методы агрегации результатов

Метод Применение Пример
Majority Voting Классификация 3 из 5 голосов за "Позитивный" → Позитивный
Среднее Числовые оценки (7 + 6 + 8) / 3 = 7
Медиана Устойчивость к выбросам [3, 7, 7, 8, 9] → 7
Weighted Voting Разная важность источников Эксперт × 2, Джун × 1
Консенсус Генерация текста Выбрать общие элементы из всех ответов

Скоринг кандидатов с Ensemble

3 промпта для оценки
Промпт 1: Общая оценка
Оцени соответствие кандидата вакансии Python Developer от 1 до 100.
[Резюме кандидата]
Оценка: 75
Промпт 2: По критериям
Оцени кандидата по критериям (1-100):
- Технические навыки: 80
- Опыт работы: 70
- Образование: 65
Средняя: 72
Промпт 3: Сравнительный
Если идеальный кандидат = 100, насколько этот кандидат соответствует требованиям?
Оценка: 68
Финальный скор:
Ensemble Score = (75 + 72 + 68) / 3 = 71.7

Интерпретация:
- 80-100: Сильный кандидат, пригласить на интервью
- 60-79: Средний кандидат, рассмотреть при нехватке
- <60: Слабое соответствие, отклонить

Результат: Средний кандидат (71.7) → Добавить в шортлист

Когда использовать Ensembling

Используйте когда:

  • Высокая цена ошибки (медицина, финансы)
  • Нужна воспроизводимость результатов
  • Задача субъективна (оценка качества)
  • Есть бюджет на дополнительные запросы
  • Результаты критически важны для бизнеса

Не используйте когда:

  • Ограниченный бюджет на API
  • Нужен быстрый ответ (latency критична)
  • Задача простая и однозначная
  • Один промпт уже даёт стабильные результаты
  • Генерация творческого контента
Анализ затрат
Подход Запросов Стоимость (×) Надёжность
Один промпт 1 1× Базовая
3 запуска одного промпта 3 3× +15-20%
3 разных промпта 3 3× +20-30%
5 запусков + 3 формулировки 15 15× +40-50%

Рекомендация: Начните с 3 разных формулировок — это оптимальный баланс между точностью и стоимостью.

Structured Outputs (2026)

📋 Гарантированный формат без примеров

В 2026 году GPT-5, Claude 4.6, Gemini 3.1 Pro поддерживают Structured Outputs — вы задаёте JSON Schema, и модель гарантированно возвращает валидный JSON. Примеры теперь нужны только для стиля и логики, а не для формата.

Раньше (до 2026)

Нужно было 3-5 примеров JSON в промпте, чтобы модель "выучила" формат. Иногда модель всё равно ломала структуру.

Примеры формата + примеры логики
= 6-10 примеров в промпте

Сейчас (2026)

JSON Schema гарантирует формат. Примеры нужны только для стиля заполнения, логики и edge cases.

Schema для формата + примеры логики
= 2-3 примера в промпте

Prompt Caching (2026)

💰 Few-shot стал в 4-10x дешевле

Все основные провайдеры в 2026 поддерживают Prompt Caching — если ваш системный промпт с примерами повторяется, вы платите за него со скидкой 75-90%.

Провайдер Скидка Как включить
Claude 4.6 90% Автоматически для промптов >1024 токенов
GPT-5 75-90% Автоматически для системных промптов
Gemini 3.1 Pro 75% Через Context Caching API
Grok 4 75% Автоматически
DeepSeek V3 90% Автоматически (disk cache)
Вывод для Few-shot: Главный аргумент против Few-shot был "дорого из-за длинных промптов". В 2026 с Prompt Caching это больше не проблема — платите полную цену только за первый запрос, потом со скидкой 75-90%.

Мультимодальный Few-shot (2026)

🖼️ Примеры с изображениями, аудио и видео

Gemini 3.1 Pro поддерживает few-shot с изображениями (до 900 в одном промпте), аудио и даже ссылками на YouTube-видео. Это открывает новые возможности:

  • Классификация изображений: "Вот 5 фото дефектов → классифицируй новое фото"
  • Извлечение данных из скриншотов: "Вот примеры парсинга → извлеки данные из нового скриншота"
  • Анализ аудио: "Вот примеры тональности звонков → оцени новый звонок"
Заметка для преподавателя

Рекомендации по проведению:

  • Покажите реальный пример: запустите один промпт 5 раз и покажите разницу в ответах
