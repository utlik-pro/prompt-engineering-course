2. "Качество отличное, но цена завышена."
3. "Внешне красиво, но функционал не соответствует описанию."
4. "Работает, но очень шумный. За эту цену можно простить."
5. "Пришло быстро, но не того цвета, что заказывал."

### 🎯 Промпт 2: Генератор SQL запросов

```
Создай SQL запрос по описанию на естественном языке.

База: PostgreSQL
Таблицы:
- users (id, name, email, created_at)
- orders (id, user_id, product, amount, order_date)
- products (id, name, price, category)

Примеры:

Описание: Найти всех пользователей, зарегистрированных в январе 2025
SQL:
SELECT * FROM users
WHERE created_at >= '2025-01-01'
  AND created_at < '2025-02-01';

Описание: Показать топ-5 товаров по количеству заказов
SQL:
SELECT products.name, COUNT(orders.id) as order_count
FROM products
JOIN orders ON products.id = orders.product
GROUP BY products.name
ORDER BY order_count DESC
LIMIT 5;

Описание: Вывести пользователей, которые не делали заказов
SQL:
SELECT users.name, users.email
FROM users
LEFT JOIN orders ON users.id = orders.user_id
WHERE orders.id IS NULL;

### 🎯 Промпт 3: Саммаризатор встреч

```
Создай краткое резюме встречи с ключевыми пунктами и action items.

Формат:
- 2-3 предложения общего резюме
- Список решений
- Action items с ответственными

Пример 1:
Транскрипт: "На встрече обсудили новый дизайн лендинга. Дизайнер Анна показала 3 варианта. Выбрали вариант B с синим фоном. Анна доделает детали к пятнице. Разработчик Иван начнет верстку в понедельник. Маркетолог Ольга подготовит тексты."

Резюме:

- Верстка начнется в понедельник

Action items:
- [ ] Анна: доделать детали дизайна (до пятницы)
- [ ] Иван: начать верстку (понедельник)
- [ ] Ольга: подготовить тексты (до понедельника)

## 5. Edge Cases для тестирования

### ⚠️ Edge Cases для классификации

```
ГРАНИЧНЫЕ СЛУЧАИ:

1. ДВУСМЫСЛЕННЫЕ:
   - "Товар неплохой, но ожидал лучшего" (позитив или негатив?)
   - "Срочно, но не критично" (срочно или важно?)
   - "Вопрос или жалоба: почему так долго?" (вопрос или жалоба?)

2. СМЕШАННЫЕ ЭМОЦИИ:
   - "Доставка супер, товар ужасный"
   - "Качество отличное, но дорого"
   - "Работает, но очень шумный"

3. САРКАЗМ:
   - "Ну просто отличный товар, сломался через день"
   - "Спасибо за 'быструю' доставку (ждал 3 недели)"

4. ВОПРОСЫ В ФОРМЕ УТВЕРЖДЕНИЙ:
   - "Интересно, когда вы исправите баг"
   - "Наверное, возврат денег невозможен"

5. НЕПОЛНАЯ ИНФОРМАЦИЯ:
   - "Хм..."
   - "Ок"
   - "Посмотрим"

6. ОПЕЧАТКИ И СЛЕНГ:
   - "Тавар норм, дастафка бистрая"
   - "топчик, рек!"
   - "Пока норм, посмотрим че дальше"
```

## 🎓 Рекомендации по использованию

### Для преподавателя:

1. **Начинайте с простых датасетов**, усложняйте постепенно
2. **Показывайте edge cases** только после базовых примеров
3. **Давайте студентам выбор** датасета (мотивация выше)
4. **Обсуждайте ошибки** — они учат больше, чем успехи

### Для студентов:

1. **Используйте эти датасеты** для тренировки
2. **Добавляйте свои примеры** из реальной работы
3. **Тестируйте на edge cases** — там модели чаще ошибаются
4. **Сохраняйте успешные промпты** в личную библиотеку
