# Модуль 4: Обучение на примерах и работа с контекстом

## 📋 Общая информация

- **Длительность:** 3 часа
- **Формат:** 1 час теории + 1.5 часа практики + 0.5 часа домашнее задание
- **Уровень:** Intermediate
- **Акцент:** МАКСИМУМ ПРАКТИКИ (70% времени)

---

## 🎯 Цели занятия

### Для студентов:
1. Понять разницу между Zero-shot, Few-shot и Many-shot prompting
2. Научиться создавать эффективные Few-shot промпты
3. Освоить подбор примеров для обучения модели
4. Применить In-Context Learning на практике
5. Создать собственные промпт-шаблоны с примерами

### Для преподавателя:
- Провести живые демонстрации на каждом инструменте
- Организовать 8-10 практических упражнений
- Помочь студентам адаптировать техники под их задачи
- Создать атмосферу экспериментирования

## 🛠 Необходимые инструменты

Убедитесь, что у студентов есть доступ к:

1. **Grok 4** (X.com / grok.com) — 2M контекст (Fast), нативный chain-of-thought, самый дешёвый API ($0.20/M токенов)
2. **ChatGPT** (GPT-5) — 400K контекст, Structured Outputs, автоматический роутинг между моделями
3. **Claude** (Opus 4.6 / Sonnet 4.6) — 200K контекст (1M в бета), авто-кэширование промптов, extended thinking
4. **Google AI Studio** (Gemini 3.1 Pro) — до 2M токенов, нативный мультимодальный few-shot (до 900 изображений)
5. **DeepSeek V3/V4** — 164K-1M контекст, открытые веса, кэширование со скидкой 75-90%
6. **Perplexity Sonar** — few-shot + веб-поиск в реальном времени, 1200 токенов/сек

**Подготовка:** За день до занятия отправьте студентам инструкцию по регистрации.

## 🎬 Сценарий начала занятия

### Первые 5 минут — Хук:

**ЧТО ГОВОРИТЬ:**

> "Представьте: вы хотите, чтобы ChatGPT писал письма клиентам в стиле вашей компании. Можно написать длинную инструкцию... А можно просто показать 3 примера. И модель поймет сама. Это называется Few-shot prompting — и это одна из самых мощных техник работы с AI."

**ЧТО ДЕЛАТЬ:**
1. Открыть ChatGPT на проекторе
2. Показать Zero-shot промпт: "Напиши письмо клиенту о задержке доставки"
3. Показать Few-shot промпт с 3 примерами
4. Показать разницу в результате

**ОЖИДАЕМАЯ РЕАКЦИЯ:** "Вау, это намного лучше!"

## 🚨 Типичные проблемы и решения

### Проблема 1: "У меня не работает Few-shot"
**Решение:** Проверить качество примеров (разнообразие, релевантность, формат)

### Проблема 2: "Слишком долго создавать примеры"
**Решение:** Показать технику "генерация примеров через AI"

### Проблема 3: "Не понимаю, сколько примеров нужно"
**Решение:** Правило: начинать с 3, увеличивать, если нужно

## 🎓 Методические рекомендации

### Для эффективной практики:

1. **Работа в парах:** Студенты обмениваются промптами и тестируют их
2. **Живое кодирование:** Преподаватель создает промпты вместе со студентами
3. **Ротация инструментов:** Каждое упражнение в новом инструменте
4. **Реальные задачи:** Последние упражнения — на задачах студентов

### Темп занятия:
- ⚡ Быстро через теорию (не застревать на слайдах)
- 🐌 Медленно через практику (давать время на эксперименты)
- 💬 Много интерактива (задавать вопросы, обсуждать результаты)
