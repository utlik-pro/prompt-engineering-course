## 📝 Задание 1: Few-shot промпт для специфичной задачи (Обязательное)

### Описание

Создайте Few-shot промпт для задачи из **вашей профессиональной сферы** или учебы.

### Требования

1. **Выберите реальную задачу**, которую вы делаете регулярно:
   - Для разработчиков: код-ревью, генерация тестов, рефакторинг
   - Для маркетологов: создание постов, email-рассылок, описаний
   - Для аналитиков: саммаризация данных, создание отчетов
   - Для дизайнеров: описание дизайн-решений, создание спецификаций
   - Для HR: описание вакансий, ответы кандидатам
   - Для студентов: конспекты, решение задач определенного типа

2. **Создайте промпт** с минимум **5 примерами**

3. **Протестируйте** на минимум **10 реальных случаях**

4. **Запишите метрики:**
   - Количество примеров: X
   - Точность: X/10 правильных
   - Время создания промпта: X минут
   - Время экономии на 10 задачах: X минут

### Шаблон выполнения

```markdown
## Задача 1: Few-shot промпт для [НАЗВАНИЕ ЗАДАЧИ]

### 1. Описание задачи
[Опишите, что делает промпт, зачем он нужен, как часто вы это делаете]

### 2. Few-shot промпт

```
[ВАШ ПРОМПТ С ПРИМЕРАМИ]
```

### 3. Тестирование

Протестировано на 10 случаях:

| № | Входные данные | Ожидаемый результат | Фактический результат | Правильно? |
|---|----------------|---------------------|----------------------|------------|
| 1 | ...            | ...                 | ...                  | ✅/❌       |
| 2 | ...            | ...                 | ...                  | ✅/❌       |
| ... | ...          | ...                 | ...                  | ✅/❌       |

**Точность:** X/10 (X%)

### 4. Метрики

- Количество примеров в промпте: X
- Время создания промпта: X минут
- Среднее время выполнения задачи вручную: X минут
- Среднее время с AI: X секунд
- Экономия времени на 10 задачах: X минут

### 5. Выводы

[Что сработало хорошо? Что можно улучшить? Будете ли использовать в работе?]
```

### Критерии оценки

- ✅ Промпт содержит минимум 5 качественных примеров
- ✅ Примеры разнообразные (покрывают разные случаи)
- ✅ Протестировано на 10 реальных случаях
- ✅ Точность > 70%
- ✅ Задача релевантна вашей работе/учебе
- ✅ Есть анализ результатов

## 📝 Задание 4: Промпт для edge cases (Дополнительное, +бонус)

### Описание

Создайте Few-shot промпт, который корректно обрабатывает сложные граничные случаи.

### Требования

1. **Выберите задачу**, где много edge cases (например, классификация смешанных эмоций, парсинг нестандартных форматов)

2. **Соберите минимум 10 edge cases**

3. **Создайте промпт**, который корректно их обрабатывает

4. **Протестируйте** на 15 случаях (10 edge cases + 5 обычных)

5. **Сравните** с базовым промптом (без edge cases)

```markdown
...
10. [Edge case 10 + почему сложный]
```

[ПРОМПТ БЕЗ EDGE CASES]
```

Точность на edge cases: X/10

### 4. Улучшенный промпт (с edge cases)

```
[ПРОМПТ С ПРИМЕРАМИ EDGE CASES]

```
| [case 2]  | ❌ [результат]  | ✅ [результат]     |
| ...       | ...            | ...               |

**Улучшение:** с X% до Y%

### 6. Выводы

[Какие edge cases важно включать? Как они улучшили промпт?]
```

### Критерии оценки (бонус +15%)

- ✅ Выбрана задача с реальными edge cases

## 💡 Советы по выполнению

### 1. Выбор задачи

- **Выбирайте реальную задачу**, которую будете использовать
- Не усложняйте без необходимости
- Если застряли, спросите у преподавателя

### 2. Создание примеров

- Используйте **реальные данные** из работы/учебы
- Включайте **разные случаи** (не все простые)
- **Проверьте качество** — нет ошибок?

### 3. Тестирование

- Тестируйте на **новых данных** (не из примеров!)
- Записывайте **все результаты** (даже ошибки)
- **Анализируйте ошибки** — почему модель ошиблась?

### 4. Улучшение

- Если точность < 70%, **добавьте примеры**
- Если модель путает категории, **добавьте граничные примеры**
- **Итерируйте** — улучшайте промпт постепенно

### 5. Документирование

- **Фиксируйте все:** промпты, результаты, метрики
- Делайте **скриншоты** удачных результатов
- **Сохраняйте промпты** — пригодятся в будущем

## ❓ FAQ

### Вопрос: Сколько времени займет домашнее задание?

**Ответ:**
- Обязательные задания (1-2): 2-3 часа
- Каждое дополнительное: +1 час

### Вопрос: Можно ли использовать любую AI-модель?

**Ответ:** Да, используйте любую доступную модель: ChatGPT (GPT-5), Claude (Opus/Sonnet 4.6), Grok 4, Gemini 3.1 Pro, DeepSeek V3/V4, Perplexity Sonar. Рекомендуем попробовать минимум 2 разные модели и сравнить результаты.

### Вопрос: Что делать, если точность < 70%?

**Ответ:**
1. Добавьте больше примеров (5 → 8 → 10)
2. Улучшите качество примеров (разнообразие, edge cases)
3. Уточните инструкцию в промпте
4. Попробуйте другую модель
5. Обратитесь к преподавателю

### Вопрос: Можно ли делать задание в команде?

**Ответ:** Можно обсуждать подходы, но промпты и тестирование — индивидуально. У каждого должна быть своя задача.

### Вопрос: Обязательно делать дополнительные задания?

**Ответ:** Нет, но они дают бонусы и углубляют понимание.

### Вопрос: Что если моя задача не подходит для Few-shot?

**Ответ:** Все задачи подходят! Если сомневаетесь, напишите преподавателю — подберем вместе.
