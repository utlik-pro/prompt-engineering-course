#### 2. Few-shot (для демонстрации улучшения)
```
Задача: Написать короткое описание продукта для маркетплейса.

Пример 1:
Продукт: Умная колонка Яндекс Станция Мини
Описание: Ваш личный помощник в компактном корпусе. Включает музыку голосом, управляет умным домом и отвечает на любые вопросы. Размером с кофейную чашку — поместится даже на узкой полке.

Пример 2:
Продукт: Механическая клавиатура Keychron K2
Описание: Почувствуйте разницу при каждом нажатии. Переключатели Gateron Brown обеспечивают тактильный отклик без лишнего шума. Работает по Bluetooth с 3 устройствами одновременно — переключайся между ноутбуком, планшетом и телефоном одной кнопкой.

Пример 3:
Продукт: Термокружка Stanley Adventure
Описание: Кофе остается горячим 6 часов, лед не тает 8 часов. Литровый объем для тех, кто пьет много. Корпус из нержавеющей стали выдержит падение с велосипеда и поездку в рюкзаке.

---

Теперь напиши описание для:
Продукт: Беспроводные наушники Sony WH-1000XM5
```

#### 3. Классификация (для практики)
```
Классифицируй запрос пользователя: Технический вопрос / Вопрос по оплате / Запрос функции / Жалоба / Общий вопрос

Примеры:

"У меня не открывается приложение после обновления" → Технический вопрос
"С карты списали деньги дважды, как вернуть?" → Вопрос по оплате
"Было бы круто, если бы можно было экспортировать в Excel" → Запрос функции
"Третий день не могу дозвониться, это издевательство!" → Жалоба
"Как изменить пароль?" → Общий вопрос

---

Классифицируй: "При попытке войти пишет 'Ошибка 500'"
```

#### 4. Извлечение данных (для демонстрации ICL)
```
Извлеки информацию из описания вакансии в JSON формат.

Пример 1:
Текст: "Компания TechCorp ищет Python разработчика с опытом от 3 лет. Зарплата 150-200k, удаленка."
JSON: {
  "company": "TechCorp",
  "position": "Python разработчик",
  "experience": "3+ лет",
  "salary": "150-200k",
  "format": "удаленка"
}

Пример 2:
Текст: "Требуется дизайнер UI/UX в офис в Минске. Опыт от года. До 100k."
JSON: {
  "company": "не указана",
  "position": "UI/UX дизайнер",
  "experience": "1+ год",
  "salary": "до 100k",
  "format": "офис, Минск"
}

---

Извлеки данные:
Текст: "StartupXYZ приглашает фулстек разработчика (React + Node.js). Опыт 2+ года. Гибрид (3 дня офис). До 180k."
```

## 🎬 Структура занятия (краткая)

### Часть 1: Теория (60 мин)

**00:00-00:15** — Введение + демо (Zero vs Few-shot)
- Открыть ChatGPT (GPT-5) или Grok 4
- Показать промпты 1 и 2 (выше)
- "Видите разницу?"

**00:15-00:30** — Zero/Few/Many-shot
- Нарисовать таблицу на доске
- Когда что использовать
- Правило: начинай с Zero, добавляй примеры
- ВАЖНО: модели 2026 требуют МЕНЬШЕ примеров (3-5 вместо 10-20)

**00:30-00:40** — In-Context Learning + Extended Thinking
- Демо промпта 4 (извлечение данных) в Claude Sonnet 4.6
- "Модель научилась формату через примеры"
- Показать: модели с extended thinking (Claude, GPT-5 Thinking) лучше рассуждают, если показать ХОД рассуждений в примерах

**00:40-00:50** — НОВОЕ: Structured Outputs + Prompt Caching
- Structured Outputs: формат больше не нужно учить через примеры
- Prompt Caching: скидка 75-90% при повторном использовании промптов
- "В 2026 few-shot стал ещё мощнее: примеры только для стиля и логики"

**00:50-01:00** — Мультимодальный few-shot + подбор примеров
- Демо в Gemini 3.1 Pro: few-shot с изображениями
- Золотые правила подбора примеров
- Плохой vs хороший набор
- Качество > количество (ключевой сдвиг 2026)

## 💬 Что говорить (key phrases)

### Начало занятия:
> "Сегодня мы работаем с моделями 2026 года — GPT-5, Claude Opus 4.6, Grok 4, Gemini 3.1 Pro. Вы научитесь учить AI через примеры, а не инструкции. И мы освоим новые техники: Structured Outputs, Prompt Caching и мультимодальный few-shot."

### После демо Zero vs Few-shot:
> "Вместо длинной инструкции — 3 примера. И модель поняла сама. Это Few-shot prompting."

### Объяснение ICL:
> "Модель 'обучается' прямо в промпте, без изменения весов. Как актер, который читает сценарий и сразу входит в роль."

### Правило количества примеров:
> "Начинайте с 3. Если результат плох — добавляйте. Если хорош — уберите лишние. Найдите баланс."

### Про edge cases:
> "Модели ошибаются не на простых случаях, а на граничных. Включайте их в примеры!"

### Мотивация для практики:
> "Следующие 90 минут — чистая практика. Создадите минимум 8 промптов. Один из них точно пригодится в работе."

### Закрытие:
> "Few-shot — это инвестиция: 10 минут на создание = часы экономии. Используйте, делитесь с коллегами, улучшайте."

## ⚠️ Частые проблемы во время занятия

### Проблема: Студенты не зарегистрированы в инструментах
**Решение:** За день до занятия отправить инструкцию с регистрацией

### Проблема: "У меня не работает Few-shot"
**Решение:**
1. Проверить качество примеров
2. Добавить больше примеров
3. Уточнить инструкцию

### Проблема: Студенты не успевают за упражнениями
**Решение:**
- Давать больше времени
- Помогать индивидуально
- Не все должны закончить все

### Проблема: Скучно только слушать теорию
**Решение:**
- Больше интерактива
- Задавать вопросы
- Просить предположить результат

### Проблема: Студенты не понимают, для чего это нужно
**Решение:**
- Больше реальных примеров
- Спросить про их задачи
- Показать экономию времени
