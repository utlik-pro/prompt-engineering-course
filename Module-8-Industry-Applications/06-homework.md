# Домашнее задание: Отраслевое применение и кейсы

## Общая информация

**Время выполнения:** 75 минут (45 минут основное задание + 30 минут дополнительное)

**Формат:** Индивидуальная работа

**Доставка:** Markdown файлы или Notion страница

---

## Задание 1: Создание отраслевой Knowledge Base (45 минут)

### Цель

Создать полноценную knowledge base для своей профессиональной области с набором специализированных промптов.

### Требования

#### Структура KB (обязательные разделы):

1. **Терминология (минимум 20 терминов)**
   - Основные термины отрасли
   - Определения
   - Примеры использования

2. **Процессы (минимум 3 процесса)**
   - Описание типичного workflow
   - Шаги выполнения
   - Чек-лист

3. **Промпты (минимум 10 промптов)**
   - Ежедневные задачи: 4 промпта
   - Сложные задачи: 3 промпта
   - Творческие задачи: 2 промпта
   - Аналитические задачи: 1 промпт

4. **Кейсы (минимум 2 кейса)**
   - Описание ситуации
   - Применённое решение
   - Результат
   - Уроки

5. **Ресурсы (минимум 10 ресурсов)**
   - Инструменты
   - Курсы
   - Сообщества

### Формат промптов в KB

Для каждого промпта укажите:

```markdown
## [Название промпта]

**Категория:** [Ежедневная/Сложная/Творческая/Аналитическая]
**Сложность:** [Базовый/Средний/Продвинутый]
**Когда использовать:** [Описание ситуации]

### Промпт:
```
[Текст промпта с плейсхолдерами [ПЕРЕМЕННАЯ]]
```

### Переменные:
- [ПЕРЕМЕННАЯ_1]: описание и пример
- [ПЕРЕМЕННАЯ_2]: описание и пример

### Пример использования:
[Конкретный пример заполненного промпта]

### Ожидаемый результат:
[Что должен вернуть AI]

### Советы:
- [Совет 1]
- [Совет 2]
```

### Структура доставки

Создайте файлы или страницы:

```
[Ваша отрасль]-KB/
├── README.md (обзор KB)
├── 01-terminology.md (глоссарий)
├── 02-processes.md (процессы и чек-листы)
├── 03-prompts/
│   ├── daily-prompts.md
│   ├── complex-prompts.md
│   ├── creative-prompts.md
│   └── analytical-prompts.md
├── 04-cases.md (кейсы)
└── 05-resources.md (ресурсы)
```

### Критерии оценки

- ✅ KB содержит все 5 обязательных разделов
- ✅ Минимум 10 промптов с полным описанием
- ✅ Минимум 20 терминов в глоссарии
- ✅ Минимум 2 документированных кейса
- ✅ Промпты протестированы и работают
- ✅ Структура логична и понятна

## Задание 2: Практическое применение (30 минут)

### Цель

Применить созданные промпты к реальной рабочей задаче и задокументировать результаты.

### Процесс

1. **Выберите реальную задачу из вашей работы**
   - Задача должна быть актуальной
   - Задача должна соответствовать одному из ваших промптов

2. **Примените промпт из KB**
   - Заполните переменные реальными данными
   - Выполните промпт в ChatGPT или Claude
   - Сохраните результат

3. **Оцените результат**
   - Качество (1-10)
   - Время выполнения
   - Необходимость доработки

4. **Улучшите промпт**
   - На основе результата
   - Добавьте уточнения
   - Обновите версию в KB

### Структура отчёта

Создайте файл `practical-application-report.md`:

