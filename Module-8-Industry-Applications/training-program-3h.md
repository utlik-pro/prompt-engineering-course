**Формат:** Теория (33%) + Практика (50%) + Домашнее задание (17%)

## БЛОК 0: Вступление (10 минут)

### Цели блока
- Познакомиться с участниками
- Определить профессиональные области участников
- Сформировать ожидания от тренинга

### Содержание

**1. Приветствие (2 минуты)**
- Представление тренера
- Краткий обзор программы
- Важность отраслевой специализации

**2. Быстрая самооценка (5 минут)**

Попросите участников ответить:
```
1. В какой отрасли вы работаете?

   - Образование/обучение

2. Используете ли вы AI в своей работе регулярно?
3. Какие задачи хотели бы автоматизировать с помощью AI?

```

**Критерии оценки:**
- Профессиональная область участника

## БЛОК 1: Теория: Отраслевое применение AI (50 минут)

### Цели блока
- Изучить применение AI в маркетинге
- Освоить AI в образовании

### 1.1 AI в маркетинге и контенте (12 минут)

**Ключевые применения:**

```
Маркетинг + AI =
- Генерация контента (посты, статьи, email)
- Копирайтинг (заголовки, CTA, описания)
- SMM (контент-планы, хештеги, анализ)
- SEO (ключевые слова, мета-описания)
- Реклама (тексты объявлений, A/B варианты)
```

**Пример промпта для маркетинга:**

```
Ты - опытный маркетолог в сфере [ОТРАСЛЬ].

Создай контент-план на неделю для Instagram аккаунта.

Целевая аудитория: [ОПИСАНИЕ ЦА]
Цель: [УВЕЛИЧЕНИЕ ПОДПИСЧИКОВ / ПРОДАЖИ / ВОВЛЕЧЕННОСТЬ]
Тон коммуникации: [ДРУЖЕЛЮБНЫЙ / ЭКСПЕРТНЫЙ / РАЗВЛЕКАТЕЛЬНЫЙ]

Для каждого дня укажи:
1. Тип контента (пост/сторис/рилс)
2. Тема и основная идея
3. Ключевые месседжи
4. Call-to-action
5. Рекомендуемые хештеги (5-7 штук)

Формат: таблица с колонками.
```

**Демонстрация (3 минуты):**
1. Откройте ChatGPT
2. Покажите генерацию контент-плана
3. Продемонстрируйте вариации для разных ниш

---

### 1.2 AI в образовании (12 минут)

**Ключевые применения:**

```
Образование + AI =
- Создание учебных материалов
- Разработка тестов и заданий
- Персонализация обучения
- Проверка и обратная связь
- Адаптивное обучение
```

**Пример промпта для образования:**

```
Ты - методист с опытом разработки образовательных программ.

Создай план урока по теме: [ТЕМА]

Параметры:
- Целевая аудитория: [ВОЗРАСТ/УРОВЕНЬ]
- Продолжительность: [ВРЕМЯ] минут
- Формат: [ОНЛАЙН/ОФЛАЙН/ГИБРИДНЫЙ]

Структура урока:
1. Введение и мотивация (цели, зачем это нужно)
2. Теоретический блок (ключевые концепции)
3. Практическое задание (упражнения)
4. Проверка понимания (мини-тест)
5. Домашнее задание
6. Дополнительные материалы

Для каждого блока укажи:
- Время
- Содержание
- Методы
- Необходимые материалы
```

**Демонстрация (3 минуты):**
1. Покажите создание плана урока
2. Продемонстрируйте генерацию теста
3. Покажите адаптацию под разные уровни

### 1.3 AI в программировании (12 минут)

**Ключевые применения:**

```
Программирование + AI =
- Написание кода
- Code review и рефакторинг
- Документирование
- Отладка и debugging
- Тестирование
```

**Пример промпта для программирования:**

```
Ты - senior developer с экспертизой в [ЯЗЫК/ФРЕЙМВОРК].

Задача: [ОПИСАНИЕ ЗАДАЧИ]

Требования:
- Язык: [Python/JavaScript/etc.]
- Стиль кода: [PEP8/Airbnb/etc.]
- Обработка ошибок: обязательно
- Комментарии: на русском языке

Предоставь:
1. Полный рабочий код
2. Комментарии к ключевым частям
3. Примеры использования
4. Возможные edge cases
5. Рекомендации по оптимизации
```

**Пример промпта для code review:**

```
Проведи code review следующего кода:

```[ЯЗЫК]
[КОД]
```

Проанализируй:
1. Качество кода (читаемость, структура)
2. Потенциальные баги
3. Производительность
4. Безопасность
5. Соответствие best practices

Формат ответа:
- Критические проблемы (нужно исправить)
- Рекомендации (желательно исправить)
- Позитивные моменты (что хорошо)
```

**Демонстрация (3 минуты):**
1. Покажите генерацию кода
2. Продемонстрируйте code review
3. Покажите создание документации

### 1.5 Создание knowledge bases (6 минут)

**Концепция Knowledge Base:**

```
Knowledge Base = структурированная база знаний,
которая позволяет AI находить и использовать
релевантную информацию для ответов
```

**Типы knowledge bases:**

1. **Персональная KB**
   - Заметки и документы
   - Личные проекты
   - Обучающие материалы

2. **Командная KB**
   - Процессы и процедуры
   - Документация проектов
   - FAQ и руководства

3. **Отраслевая KB**
   - Терминология и определения
   - Best practices
   - Кейсы и примеры

**Структура отраслевой KB:**

```
[ОТРАСЛЬ] Knowledge Base/
├── Терминология/
│   ├── Основные термины
│   ├── Глоссарий
│   └── Аббревиатуры
├── Процессы/
│   ├── Типичные workflow
│   ├── Чек-листы
│   └── Шаблоны
├── Промпты/
│   ├── Базовые промпты
│   ├── Продвинутые промпты
│   └── Специализированные промпты
├── Кейсы/
│   ├── Успешные примеры
│   ├── Ошибки и уроки
│   └── Best practices
└── Ресурсы/
    ├── Инструменты
    ├── Курсы
    └── Сообщества
```

### 5.2 Итоги и обсуждение (10 минут)

**Ключевые выводы:**

**Маркетинг:**
- AI генерирует контент, но нужна редактура
- Важно задавать контекст и тон коммуникации
- A/B варианты помогают найти лучшее решение

**Образование:**
- AI создает структуру, человек добавляет экспертизу
- Персонализация требует детального описания аудитории
- Тесты нужно проверять на корректность

**Программирование:**
- AI ускоряет написание кода, но код нужно проверять
- Документация и тесты создаются быстрее
- Code review выявляет проблемы, которые человек может пропустить

**Аналитика:**
- AI помогает структурировать анализ
- Важно правильно формулировать вопросы
- Выводы требуют экспертной валидации
