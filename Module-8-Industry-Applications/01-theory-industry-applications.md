# Теория: Отраслевое применение AI

## Введение

AI-инструменты трансформируют рабочие процессы во всех отраслях. Понимание специфики применения промпт-инжиниринга в каждой сфере позволяет максимизировать эффективность и создавать реальную ценность.

---

## 1. AI в маркетинге и контенте

### 1.1 Обзор применений

```
Маркетинг + AI =
├── Контент-маркетинг
│   ├── Статьи и блог-посты
│   ├── Social media контент
│   └── Email-маркетинг
├── Копирайтинг
│   ├── Рекламные тексты
│   ├── Лендинги
│   └── Продуктовые описания
├── SMM
│   ├── Контент-планы
│   ├── Хештеги и теги
│   └── Комьюнити-менеджмент
├── SEO
│   ├── Ключевые слова
│   ├── Мета-описания
│   └── Оптимизация контента
└── Реклама
    ├── Тексты объявлений
    ├── A/B варианты
    └── Таргетинг
```

### 1.2 Ключевые промпты для маркетинга

#### Генерация контент-плана

```
Ты - контент-стратег с 10-летним опытом в digital-маркетинге.

Создай контент-план на месяц для [БРЕНД/КОМПАНИЯ].

Контекст:
- Ниша: [НИША]
- Целевая аудитория: [ОПИСАНИЕ ЦА]
- Цели: [ЦЕЛИ МАРКЕТИНГА]
- Платформы: [Instagram/Facebook/LinkedIn/etc.]
- Тон голоса: [ОПИСАНИЕ ТОНА]

Для каждой недели укажи:
- Темы постов (3-4 поста)
- Типы контента (карусель/рилс/сторис/пост)
- Ключевые сообщения
- CTA для каждого поста
- Рекомендуемое время публикации
- Хештеги (5-7 релевантных)

Формат: таблица по неделям.
```

#### Создание рекламного текста

```
Напиши рекламный текст для [ПРОДУКТ/УСЛУГА].

Параметры:
- Формат: [Google Ads / Facebook Ads / Instagram]
- Цель: [Конверсия / Трафик / Осведомленность]
- Целевая аудитория: [ОПИСАНИЕ]
- УТП: [УНИКАЛЬНОЕ ТОРГОВОЕ ПРЕДЛОЖЕНИЕ]
- Ограничения: [ДЛИНА / ЗАПРЕЩЕННЫЕ СЛОВА]

Создай 5 вариантов с разными подходами:
1. Эмоциональный (на боли и желания)
2. Рациональный (факты и цифры)
3. Социальное доказательство (отзывы, кейсы)
4. Срочность (ограниченное предложение)
5. Вопросительный (вовлекающие вопросы)

Для каждого варианта:
- Заголовок
- Основной текст
- CTA
- Рекомендация по визуалу
```

#### Email-рассылка

```
Создай email-рассылку для [ЦЕЛЬ РАССЫЛКИ].

Контекст:
- Бренд: [НАЗВАНИЕ]
- Аудитория: [ОПИСАНИЕ]
- Тип письма: [Welcome/Продающее/Информационное/Реактивация]
- Предложение: [ЧТО ПРЕДЛАГАЕМ]

Структура:
1. Тема письма (3 варианта, до 50 символов)
2. Прехедер (до 100 символов)
3. Приветствие
4. Основной блок (проблема → решение → выгоды)
5. CTA (кнопка)
6. P.S. (дополнительный крючок)
7. Подпись

Дополнительно:
- Варианты для A/B теста темы
- Рекомендации по времени отправки
```

### 1.3 Best practices для маркетинга

**Генерация контента:**
- Всегда указывайте целевую аудиторию
- Задавайте тон голоса бренда
- Просите несколько вариантов для A/B тестирования
- Редактируйте результат под свой стиль

**Ограничения:**
- AI может не знать актуальные тренды
- Проверяйте факты и цифры
- Добавляйте уникальность бренда вручную
- Не публикуйте без редактуры

## 2. AI в образовании

### 2.1 Обзор применений

```
Образование + AI =
├── Создание материалов
│   ├── Планы уроков
│   ├── Презентации
│   └── Раздаточные материалы
├── Оценивание
│   ├── Тесты и квизы
│   ├── Задания
│   └── Критерии оценки
├── Персонализация
│   ├── Адаптивные материалы
│   ├── Индивидуальные траектории
│   └── Рекомендации
├── Обратная связь
│   ├── Проверка работ
│   ├── Комментарии
│   └── Рекомендации по улучшению
└── Администрирование
    ├── Отчеты
    ├── Планирование
    └── Коммуникация
```

### 2.2 Ключевые промпты для образования

#### Создание плана урока

```
Ты - методист с опытом в [ПРЕДМЕТНАЯ ОБЛАСТЬ].

Создай детальный план урока.

Параметры:
- Тема: [ТЕМА УРОКА]
- Предмет: [ПРЕДМЕТ]
- Класс/уровень: [КЛАСС ИЛИ УРОВЕНЬ]
- Продолжительность: [ВРЕМЯ] минут
- Формат: [Очный/Онлайн/Гибридный]
- Количество учеников: [ЧИСЛО]

Структура плана:

1. ЦЕЛИ УРОКА
   - Образовательные (знания)
   - Развивающие (навыки)
   - Воспитательные (ценности)

2. НЕОБХОДИМЫЕ МАТЕРИАЛЫ
   - Оборудование
   - Раздаточные материалы
   - Цифровые ресурсы

3. ХОД УРОКА
   [Для каждого этапа: время, деятельность учителя,
   деятельность учеников, материалы]

   - Организационный момент (X мин)
   - Актуализация знаний (X мин)
   - Изучение нового материала (X мин)
   - Первичное закрепление (X мин)
   - Практическая работа (X мин)
   - Рефлексия (X мин)
   - Домашнее задание (X мин)

4. ДИФФЕРЕНЦИАЦИЯ
   - Для слабых учеников
   - Для сильных учеников
   - Для учеников с особыми потребностями

5. ОЦЕНИВАНИЕ
   - Формативное (в процессе)
   - Суммативное (итоговое)
   - Критерии оценки

6. РЕФЛЕКСИЯ УЧИТЕЛЯ
   - Вопросы для самоанализа
```

#### Создание теста

```
Создай тест по теме: [ТЕМА]

Параметры:
- Предмет: [ПРЕДМЕТ]
- Уровень: [КЛАСС/КУРС]
- Количество вопросов: [ЧИСЛО]
- Время выполнения: [МИНУТЫ]
- Цель: [Проверка знаний/Диагностика/Итоговый]

Распределение по уровням сложности:
- Базовый (знание, понимание): 40%
- Средний (применение, анализ): 40%
- Продвинутый (синтез, оценка): 20%

Типы вопросов:
- Выбор одного ответа: [ЧИСЛО] вопросов
- Множественный выбор: [ЧИСЛО] вопросов
- Соответствие: [ЧИСЛО] вопросов
- Последовательность: [ЧИСЛО] вопросов
- Открытый ответ: [ЧИСЛО] вопросов

Для каждого вопроса укажи:
1. Номер и тип вопроса
2. Формулировка вопроса
3. Варианты ответов (где применимо)
4. Правильный ответ
5. Баллы за вопрос
6. Объяснение (для разбора)
7. К какому уровню относится

В конце предоставь:
- Ключи для проверки
- Шкалу оценивания
- Спецификацию теста
```

#### Адаптация материала

```
Адаптируй следующий учебный материал для [АУДИТОРИЯ]:

Исходный материал:
[ТЕКСТ ИЛИ ОПИСАНИЕ МАТЕРИАЛА]

Параметры адаптации:
- Возраст/уровень: [ОПИСАНИЕ]
- Особенности аудитории: [ОСОБЕННОСТИ]
- Цель адаптации: [УПРОЩЕНИЕ/УГЛУБЛЕНИЕ/ВИЗУАЛИЗАЦИЯ]
- Формат результата: [ТЕКСТ/СХЕМА/ЗАДАНИЯ]

Требования:
1. Сохранить ключевые концепции
2. Использовать понятный язык для аудитории
3. Добавить примеры из жизни аудитории
4. Включить визуальные элементы (описание)
5. Предложить интерактивные элементы

Результат должен включать:
- Адаптированный текст
- Глоссарий новых терминов
- 3 вопроса для проверки понимания
- Практическое задание
```

### 2.3 Best practices для образования

**Создание материалов:**
- Указывайте конкретный возраст и уровень
- Просите дифференциацию для разных групп
- Включайте интерактивные элементы
- Проверяйте педагогическую корректность

**Оценивание:**
- Балансируйте типы вопросов
- Включайте разные уровни сложности
- Всегда добавляйте объяснения к ответам
- Тестируйте вопросы перед использованием

**Ограничения:**
- AI не заменяет экспертизу педагога
- Материалы требуют адаптации под конкретный класс
- Проверяйте фактическую корректность
- Учитывайте культурный контекст

## 3. AI в программировании

### 3.1 Обзор применений

```
Программирование + AI =
├── Написание кода
│   ├── Функции и модули
│   ├── Классы и объекты
│   └── Интеграции и API
├── Code Review
│   ├── Анализ качества
│   ├── Поиск багов
│   └── Рекомендации
├── Рефакторинг
│   ├── Оптимизация
│   ├── Улучшение читаемости
│   └── Модернизация
├── Документация
│   ├── Docstrings
│   ├── README
│   └── API документация
├── Тестирование
│   ├── Unit-тесты
│   ├── Интеграционные тесты
│   └── Test cases
└── Отладка
    ├── Анализ ошибок
    ├── Debugging
    └── Исправление багов
```

### 3.2 Ключевые промпты для программирования

#### Написание функции

```
Напиши функцию на [ЯЗЫК] для [ЗАДАЧА].

Контекст:
- Проект: [ТИП ПРОЕКТА]
- Версия языка: [ВЕРСИЯ]
- Фреймворк: [ФРЕЙМВОРК, если есть]
- Стиль кода: [PEP8/Airbnb/Google/etc.]

Требования:
- Входные параметры: [ОПИСАНИЕ]
- Возвращаемое значение: [ОПИСАНИЕ]
- Обработка ошибок: [ТРЕБОВАНИЯ]
- Производительность: [ТРЕБОВАНИЯ]
- Совместимость: [ТРЕБОВАНИЯ]

Функция должна:
1. [ТРЕБОВАНИЕ 1]
2. [ТРЕБОВАНИЕ 2]
3. [ТРЕБОВАНИЕ 3]

Предоставь:
1. Код функции с docstring
2. Типизацию (type hints)
3. Обработку edge cases
4. 3 примера использования
5. Unit-тесты (минимум 5)

Дополнительно:
- Оценка временной сложности
- Возможные улучшения
- Альтернативные подходы
```

#### Code Review

```
Проведи детальный code review следующего кода:

```[ЯЗЫК]
[КОД]
```

Контекст:
- Назначение кода: [ОПИСАНИЕ]
- Уровень критичности: [Высокий/Средний/Низкий]
- Стандарты: [СТАНДАРТЫ КОДА]

Проанализируй по категориям:

1. КОРРЕКТНОСТЬ
   - Логические ошибки
   - Edge cases
   - Потенциальные баги

2. ЧИТАЕМОСТЬ
   - Именование переменных
   - Структура кода
   - Комментарии

3. ПРОИЗВОДИТЕЛЬНОСТЬ
   - Временная сложность
   - Пространственная сложность
   - Узкие места

4. БЕЗОПАСНОСТЬ
   - Уязвимости
   - Валидация входных данных
   - Защита данных

5. BEST PRACTICES
   - Соответствие стандартам
   - DRY, SOLID, KISS
   - Современные подходы

6. ТЕСТИРУЕМОСТЬ
   - Возможность тестирования
   - Зависимости
   - Изоляция

Формат ответа:
- Критические (необходимо исправить)
- Важные (рекомендуется исправить)
- Незначительные (можно улучшить)
- Положительные моменты

Для каждого замечания:
- Строка кода
- Проблема
- Рекомендация
- Пример исправления
```

#### Создание документации

```
Создай документацию для следующего кода:

```[ЯЗЫК]
[КОД]
```

Тип документации: [README/API Docs/Inline Comments/All]

Создай:

1. README.md
   - Описание проекта/модуля
   - Установка
   - Быстрый старт
   - API Reference (краткий)
   - Примеры использования
   - Зависимости
   - Лицензия

2. API Документация
   - Описание каждой функции/класса
   - Параметры с типами
   - Возвращаемые значения
   - Исключения
   - Примеры

3. Inline комментарии
   - Docstrings для всех public методов
   - Комментарии к сложной логике
   - TODO для улучшений

Стиль документации: [Google/NumPy/Sphinx/JSDoc]
Язык документации: [Русский/Английский]
```

### 3.3 Best practices для программирования

**Написание кода:**
- Указывайте версию языка и зависимости
- Описывайте контекст использования
- Просите обработку ошибок
- Всегда запрашивайте тесты

**Code Review:**
- Предоставляйте контекст кода
- Указывайте критичность проекта
- Просите конкретные примеры исправлений
- Используйте как второе мнение, не единственное

**Ограничения:**
- AI может генерировать нерабочий код
- Всегда тестируйте сгенерированный код
- Проверяйте безопасность
- Не доверяйте слепо рекомендациям

## 4. AI в аналитике

### 4.1 Обзор применений

```
Аналитика + AI =
├── Анализ данных
│   ├── Exploratory Data Analysis
│   ├── Статистический анализ
│   └── Визуализация
├── Отчетность
│   ├── Дашборды
│   ├── Регулярные отчеты
│   └── Ad-hoc анализ
├── Прогнозирование
│   ├── Тренды
│   ├── Forecasting
│   └── Сценарии
├── Бизнес-аналитика
│   ├── KPI анализ
│   ├── Конкурентный анализ
│   └── Market research
└── Data Science
    ├── Feature engineering
    ├── Модели ML
    └── Интерпретация
```

### 4.2 Ключевые промпты для аналитики

#### Анализ данных

```
Проведи анализ следующих данных:

Данные:
[ДАННЫЕ ИЛИ ОПИСАНИЕ ДАННЫХ]

Контекст:
- Бизнес-домен: [СФЕРА]
- Цель анализа: [ЦЕЛЬ]
- Аудитория отчета: [КТО БУДЕТ ЧИТАТЬ]
- Период: [ВРЕМЕННОЙ ПЕРИОД]

Проведи анализ:

1. ОПИСАТЕЛЬНАЯ СТАТИСТИКА
   - Основные метрики (mean, median, std)
   - Распределение значений
   - Выбросы и аномалии

2. ТРЕНДЫ И ПАТТЕРНЫ
   - Временные тренды
   - Сезонность
   - Корреляции

3. СЕГМЕНТАЦИЯ
   - Ключевые сегменты
   - Различия между группами
   - Характеристики сегментов

4. ИНСАЙТЫ
   - Ключевые находки (топ-5)
   - Неожиданные результаты
   - Гипотезы для проверки

5. РЕКОМЕНДАЦИИ
   - Действия на основе данных
   - Приоритеты
   - Метрики для отслеживания

6. ВИЗУАЛИЗАЦИЯ
   - Какие графики нужны
   - Описание каждой визуализации
   - Ключевые выводы на графиках

Формат: структурированный отчет.
```

#### Создание отчета

```
Создай аналитический отчет.

Тип отчета: [Еженедельный/Месячный/Квартальный/Ad-hoc]
Тема: [ТЕМА ОТЧЕТА]
Данные: [ОПИСАНИЕ ДАННЫХ]

Структура отчета:

1. EXECUTIVE SUMMARY
   - Ключевые выводы (3-5 пунктов)
   - Критические метрики
   - Главная рекомендация

2. МЕТОДОЛОГИЯ
   - Источники данных
   - Период анализа
   - Ограничения

3. КЛЮЧЕВЫЕ ПОКАЗАТЕЛИ
   - KPI и их динамика
   - Сравнение с целями
   - Сравнение с прошлым периодом

4. ДЕТАЛЬНЫЙ АНАЛИЗ
   - По категориям/сегментам
   - Причины изменений
   - Контекст

5. ВИЗУАЛИЗАЦИИ
   - Описание графиков
   - Интерпретация

6. ВЫВОДЫ И РЕКОМЕНДАЦИИ
   - Проблемы и возможности
   - Конкретные действия
   - Ожидаемый эффект

7. СЛЕДУЮЩИЕ ШАГИ
   - План действий
   - Ответственные
   - Сроки

Аудитория: [РУКОВОДСТВО/КОМАНДА/КЛИЕНТЫ]
Тон: [ФОРМАЛЬНЫЙ/НЕФОРМАЛЬНЫЙ]
Длина: [СТРАНИЦЫ ИЛИ СЛОВА]
```

#### SQL-запросы

```
Напиши SQL-запрос для [ЗАДАЧА].

Контекст:
- СУБД: [PostgreSQL/MySQL/BigQuery/etc.]
- Схема данных:
  [ОПИСАНИЕ ТАБЛИЦ И СВЯЗЕЙ]

Требования:
- Фильтры: [УСЛОВИЯ]
- Группировка: [ПО КАКИМ ПОЛЯМ]
- Сортировка: [ПОРЯДОК]
- Ограничения: [ЛИМИТЫ]

Дополнительно:
- Оптимизируй для производительности
- Добавь комментарии к сложным частям
- Предложи индексы если нужно
- Укажи ожидаемое время выполнения
```

### 4.3 Best practices для аналитики

**Анализ данных:**
- Описывайте контекст и бизнес-цели
- Указывайте ограничения данных
- Просите несколько интерпретаций
- Валидируйте инсайты экспертно

**Отчетность:**
- Адаптируйте под аудиторию
- Фокусируйтесь на actionable insights
- Визуализации описывайте словами
- Проверяйте расчеты

**Ограничения:**
- AI не имеет доступа к реальным данным
- Интерпретации требуют валидации
- SQL-запросы нужно тестировать
- Выводы зависят от качества входных данных

## 5. Общие принципы отраслевого применения

### 5.1 Формула эффективного промпта

```
ОТРАСЛЕВОЙ ПРОМПТ =
  РОЛЬ (отраслевой эксперт) +
  КОНТЕКСТ (специфика отрасли) +
  ЗАДАЧА (конкретное действие) +
  ФОРМАТ (отраслевой стандарт) +
  ОГРАНИЧЕНИЯ (требования индустрии)
```

### 5.2 Адаптация под отрасль

1. **Изучите терминологию** - используйте профессиональный язык
2. **Поймите процессы** - встраивайте AI в существующие workflow
3. **Определите стандарты** - соблюдайте отраслевые требования
4. **Учитывайте ограничения** - compliance, конфиденциальность
5. **Тестируйте результаты** - валидация экспертами отрасли

### 5.3 Типичные ошибки

**Избегайте:**
- Слишком общих промптов без отраслевой специфики
- Игнорирования терминологии и стандартов
- Использования результатов без проверки
- Переоценки возможностей AI
- Недооценки необходимости редактуры

3. Постоянной валидации результатов
