## 1.3 Генерация изображений через промпты (15 минут)

### Что говорить преподавателю:

**Введение:**
"Теперь перейдем к обратной стороне - не анализу, а созданию изображений. Современные модели позволяют генерировать визуальный контент просто через текстовое описание. Это революция для маркетинга, дизайна, контента."

**Основные инструменты:**
"Сегодня мы будем работать с:
- **ChatGPT (DALL-E 3/4)** - интегрирован прямо в интерфейс
- **Nanobanana** - специализированный инструмент для быстрой генерации
- **GPT-5 image generation** (если доступен) - следующее поколение

Каждый инструмент имеет свои особенности, и мы разберемся, когда использовать какой."

**Принципы создания промптов для изображений:**

"Генерация изображений - это отдельное искусство. Вот ключевые принципы:

**1. Структура промпта:**
```
[Субъект] + [Действие] + [Контекст] + [Стиль] + [Технические параметры]
```

Пример: 'Профессиональный бизнесмен в костюме, работающий за современным ноутбуком в светлом офисе с панорамными окнами, фотореалистичный стиль, естественное освещение, 16:9'

**2. Стили и референсы:**
- Фотореалистичный: 'photorealistic, high quality, professional photography'
- Иллюстрация: 'digital illustration, flat design, vector art'
- Художественный: 'oil painting, watercolor, impressionist style'
- 3D: '3D render, octane render, unreal engine'

**3. Композиция и ракурс:**
- 'close-up shot' - крупный план
- 'wide angle' - широкий угол
- 'top-down view' - вид сверху
- 'isometric view' - изометрия

**4. Освещение и атмосфера:**
- 'golden hour lighting' - золотой час
- 'dramatic lighting' - драматическое освещение
- 'soft natural light' - мягкий естественный свет
- 'neon lighting' - неоновое освещение

**5. Технические детали:**
- Соотношение сторон: '16:9', 'square', '9:16' (вертикально)
- Качество: 'high quality', '4K', 'detailed'
- Настроение: 'professional', 'cozy', 'futuristic', 'minimalist'"

**Практические примеры промптов:**

"Давайте разберем конкретные сценарии:

**Для социальных сетей (Instagram):**
```
Современный flat lay композиция с ноутбуком, кофе,
блокнотом и растением на белом мраморном столе,
минималистичный стиль, мягкое естественное освещение
сверху, пастельные тона, квадратный формат
```

**Для презентаций:**
```
Изометрическая 3D иллюстрация облачной инфраструктуры
с серверами, базами данных и пользователями,
современный корпоративный стиль, синие и голубые тона,
чистый фон, 16:9
```

**Для маркетинга:**
```
Профессиональная фотография продукта - беспроводные
наушники на градиентном фоне от темно-синего к фиолетовому,
студийное освещение с мягкими тенями, премиальный вид,
крупный план, вертикальный формат для сторис
```

**Для контента блога:**
```
Концептуальная иллюстрация искусственного интеллекта
и человека, работающих вместе, современный цифровой арт,
теплые и холодные тона, символизирующие синергию технологий
и человечности, горизонтальный формат
```"

**Типичные ошибки:**
"Что НЕ нужно делать:
- ❌ Слишком короткие промпты: 'красивый пейзаж'
- ❌ Противоречивые требования: 'реалистичный мультяшный стиль'
- ❌ Слишком сложные сцены с множеством объектов
- ❌ Использование брендированных названий (Nike, Apple)
- ❌ Просьба сгенерировать текст на изображении (плохо получается)

✅ Вместо этого:
- Детальное описание с конкретными характеристиками
- Четкий выбор одного стиля
- Фокус на 1-3 главных элементах
- Общие описания ('спортивная обувь', 'современный смартфон')
- Добавление текста отдельно в графическом редакторе"

## 1.4 Вайбкодинг и будущее мультимодальности (15 минут)

### Что говорить преподавателю:

**Введение в вайбкодинг:**
"А теперь самое интересное - вайбкодинг, или 'coding by vibe'. Это новый подход к созданию приложений, где вы описываете, что хотите получить, возможно, показываете референс - и ИИ создает готовый код и интерфейс. Это не просто генерация кода - это полноценная разработка через естественное общение."

**Основные платформы:**

"Мы будем работать с тремя главными инструментами:

**1. v0.dev от Vercel:**
- Специализируется на веб-компонентах и UI
- Генерирует React/Next.js код
- Интерактивный предпросмотр в реальном времени
- Итеративная разработка - можно уточнять и улучшать

*Когда использовать:* Создание лендингов, интерфейсов, компонентов

**2. Lovable.dev (formerly GPT Engineer):**
- Создает полноценные приложения
- Поддерживает фронтенд и бэкенд
- Интеграция с базами данных
- Деплой одним кликом

*Когда использовать:* Полноценные web-приложения, MVP стартапов

**3. Claude.ai с Artifacts:**
- Универсальный инструмент
- Генерирует интерактивные компоненты прямо в чате
- Поддержка React, HTML, SVG
- Отлично для прототипирования

*Когда использовать:* Быстрые прототипы, дашборды, визуализации

**Дополнительно (упомянуть):**
- **Cursor.ai** - IDE с встроенным ИИ для профессиональной разработки
- **Replit Agent** - для создания полноценных приложений с деплоем
- **Bolt.new** - быстрое прототипирование fullstack приложений"

**Принципы вайбкодинга:**

"Ключевые правила успешного вайбкодинга:

**1. Начните с визуального референса:**
- Покажите скриншот похожего сайта/приложения
- Опишите конкретные элементы, которые нравятся
- Укажите цветовую схему и стиль

Пример: 'Создай лендинг в стиле Apple.com - минималистичный, много воздуха,
крупные изображения продукта, плавные анимации при скролле'

**2. Итеративный подход:**
- Сначала создайте базовую структуру
- Затем уточняйте детали по частям
- Не пытайтесь описать все сразу

Диалог:
- 'Создай лендинг для AI-сервиса с хиро-секцией, преимуществами и формой'
- 'Сделай хиро-секцию более яркой, добавь градиент'
- 'Измени форму - сделай ее компактной справа'

**3. Конкретные требования:**
- Укажите технологии: 'используй Tailwind CSS'
- Определите layout: 'адаптивный дизайн, mobile-first'
- Добавьте функциональность: 'форма должна отправлять email'

**4. Используйте комбинированные промпты:**
- Текст + скриншот референса
- Описание + схема структуры
- Требования + примеры кода

Пример: 'Вот скриншот дизайна. Создай такую же карточку продукта,
но измени цвета на синие оттенки и добавь анимацию при наведении'"

**Практический workflow:**

"Типичный процесс вайбкодинга выглядит так:

**Этап 1: Идея и референсы (5 минут)**
- Соберите скриншоты сайтов, которые нравятся
- Определите ключевые элементы
- Набросайте структуру на бумаге

**Этап 2: Первая генерация (5-10 минут)**
- Опишите общую концепцию платформе
- Получите базовую версию
- Оцените, что близко к желаемому

**Этап 3: Итерации (15-30 минут)**
- Уточняйте каждую секцию
- Исправляйте детали
- Добавляйте функциональность

**Этап 4: Финализация (10 минут)**
- Проверка на мобильных устройствах
- Оптимизация производительности
- Деплой

Итого: за 30-60 минут можно создать полноценный лендинг!"

### Упражнение 1: Извлечение данных из таблиц (10 минут)

**Инструкции преподавателя:**

"**Задача:** Вы получили скриншот таблицы с финансовыми данными. Нужно извлечь данные и преобразовать в удобный формат.

**Шаг 1: Подготовка**
Откройте [этот Google Sheet](https://docs.google.com/spreadsheets/d/example) или используйте любую сложную таблицу Excel/Google Sheets. Сделайте скриншот.

**Шаг 2: Базовое извлечение**
Загрузите скриншот в ChatGPT и используйте промпт:
```
Извлеки данные из этой таблицы и преобразуй в формат JSON.
Сохрани структуру: месяц, доход, расходы, прибыль.
```

**Шаг 3: Расширенный анализ**
Теперь усложним задачу:
```
Проанализируй данные из таблицы:
1. Извлеки все числовые данные
2. Рассчитай средние значения по каждой колонке
3. Определи месяц с максимальной прибылью
4. Создай краткий текстовый отчет с выводами
5. Предложи формат Excel-формул для автоматизации расчетов
```

**Шаг 4: Сравнение моделей**
Проделайте то же самое в Claude и Gemini. Обратите внимание:
- Какая модель точнее извлекла данные?
- Где лучше форматирование ответа?
- Кто дал более полезные инсайты?"

**Что ожидать от студентов:**
- Все должны успешно извлечь базовые данные
- Некоторые обнаружат, что Claude лучше форматирует JSON
- Gemini может быть точнее с числами

**Troubleshooting:**
- Если модель неправильно распознала числа → попросите проверить еще раз
- Если формат ответа неудобный → уточните формат в промпте
- Если таблица слишком большая → разбейте на части

### Упражнение 2: Анализ UI/UX дизайна (10 минут)

**Инструкции преподавателя:**

"**Задача:** Проанализировать скриншот лендинга и получить профессиональный UX-аудит.

**Шаг 1: Выбор объекта**
Откройте любой лендинг (можете использовать свой проект или взять популярный сайт). Сделайте full-page скриншот (можно использовать расширение GoFullPage для Chrome).

**Шаг 2: Базовый анализ**
Промпт для ChatGPT/Claude:
```
Проанализируй UX/UI дизайн этого лендинга. Оцени:
1. Визуальную иерархию
2. Читабельность текста
3. CTA (call-to-action) элементы
4. Мобильную адаптивность (на основе desktop версии)
5. Общее впечатление

Дай оценку по 10-балльной шкале и краткие комментарии.
```

**Шаг 3: Детальный аудит**
Теперь получим конкретные рекомендации:
```
На основе предыдущего анализа, предложи:
1. 5 конкретных улучшений UX (с обоснованием каждого)
2. Проблемы accessibility (доступность для людей с ограничениями)
3. Что можно улучшить в цветовой схеме
4. Рекомендации по типографике
5. Где добавить/убрать whitespace

```

**Шаг 4: Сравнение с best practices**
```

```

### Частые вопросы студентов и ответы:

**Q: Почему модель неправильно распознала текст на изображении?**
A: Несколько причин:
- Низкое качество изображения → попробуйте лучшее разрешение

- Попробуйте другую модель (Gemini часто лучше с текстом)

**Q: Генерация изображений не дает того, что я хочу**
A: Чеклист:
- Промпт достаточно детальный? (минимум 50 слов)
- Указали стиль? ('photorealistic', 'illustration', etc.)
- Уточнили композицию? ('close-up', 'wide angle')
- Описали освещение и цвета?
- Попробуйте добавить '4K, detailed, high quality'

**Q: v0.dev сгенерировал код с ошибками**
A: Это нормально на ранних стадиях:
- Опишите ошибку в следующем промпте
- Попросите исправить конкретный компонент
- Начните заново с более четким промптом
- Экспортируйте код и исправьте локально

**Q: Как деплоить лендинг?**
A:
- **v0.dev:** кнопка 'Deploy' → автоматически на Vercel
- **Lovable:** встроенный deploy, кнопка в интерфейсе
- **Claude:** скопировать код → создать проект на Vercel/Netlify → загрузить код

**Q: Сколько стоят эти инструменты?**
A:
- **ChatGPT:** $20/месяц (Plus) для DALL-E 3
- **Claude:** $20/месяц (Pro) для большего лимита
- **Gemini:** бесплатная версия доступна
- **v0.dev:** бесплатные credits, потом $20/месяц
