Structured Outputs

JSON mode, схемы данных и получение структурированных ответов от AI

Цели урока

Что такое структурированный вывод?

Structured Output — это ответ AI в заранее определённом формате (JSON, XML, CSV), который можно легко парсить и использовать в других приложениях.
Обычный ответ:

Компания Apple была основана в 1976 году Стивом Джобсом, Стивом Возняком и Рональдом Уэйном. Штаб-квартира находится в Купертино, Калифорния.

Структурированный ответ: 
{
  "company": "Apple",
  "founded": 1976,
  "founders": ["Стив Джобс", "Стив Возняк", "Рональд Уэйн"],
  "headquarters": "Купертино, CA"
}

Зачем нужен структурированный вывод?

1.
Интеграция с приложениями

JSON легко парсится и используется в коде

2.
Автоматизация

Можно строить пайплайны обработки данных

3.
Предсказуемость

Всегда получаете данные в ожидаемом формате

4.
Валидация

Можно проверять корректность ответа по схеме

JSON Mode в AI-платформах

Заметки для преподавателя

Демонстрация: Покажите разницу между обычным промптом и промптом с запросом JSON. Студенты часто удивляются, насколько чётко AI следует схеме.

Как запросить JSON
Простой запрос JSON 
Проанализируй текст и верни результат в формате JSON.

Текст: "iPhone 15 Pro Max стоит 159 990 рублей и имеет
камеру 48 МП с 5x оптическим зумом."

Ответь ТОЛЬКО валидным JSON, без пояснений.

AI сам определит структуру. Подходит для простых задач.

Запрос с описанием схемы 
Проанализируй текст и верни JSON со следующей структурой:

{
  "product_name": "название товара (string)",
  "price": цена в рублях (number),
  "currency": "RUB",
  "features": ["массив ключевых характеристик"]
}

Текст: "iPhone 15 Pro Max стоит 159 990 рублей и имеет
камеру 48 МП с 5x оптическим зумом."

Ответь ТОЛЬКО валидным JSON.

Вы контролируете структуру. Лучший вариант для большинства задач.

Запрос с примером 
Проанализируй текст и верни JSON.

Пример вывода:
{
  "product_name": "Samsung Galaxy S24",
  "price": 89990,
  "currency": "RUB",
  "features": ["камера 50 МП", "8K видео"]
}

Текст для анализа: "iPhone 15 Pro Max стоит 159 990 рублей
и имеет камеру 48 МП с 5x оптическим зумом."

Ответь в таком же формате JSON.

Few-shot подход. AI точно следует примеру.

Практические примеры

Извлечение данных из текста
Извлечение контактов 
Извлеки контактную информацию из текста и верни JSON:

{
  "name": "имя контакта",
  "email": "email или null",
  "phone": "телефон или null",
  "company": "компания или null",
  "position": "должность или null"
}

Текст:
"Добрый день! Меня зовут Александр Петров, я руководитель
отдела маркетинга в компании TechSolutions. Можете связаться
со мной по email: a.petrov@techsolutions.ru или по телефону
+7 (495) 123-45-67."

Верни ТОЛЬКО JSON.
Ожидаемый результат: 
{
  "name": "Александр Петров",
  "email": "a.petrov@techsolutions.ru",
  "phone": "+7 (495) 123-45-67",
  "company": "TechSolutions",
  "position": "руководитель отдела маркетинга"
}
Анализ отзывов
Анализ отзыва клиента 
Проанализируй отзыв клиента и верни JSON:

{
  "sentiment": "positive" | "negative" | "neutral",
  "score": число от 1 до 5,
  "topics": ["список затронутых тем"],
  "pros": ["плюсы, если упомянуты"],
  "cons": ["минусы, если упомянуты"],
  "action_required": true | false,
  "summary": "краткое резюме в 1 предложение"
}

Отзыв:
"Заказывал диван, доставили через 3 недели вместо обещанных 5 дней!
Качество самого дивана отличное, мягкий и удобный. Но сервис
ужасный — никто не предупредил о задержке, на звонки не отвечали."

Верни ТОЛЬКО JSON.
Генерация структурированного контента
Генерация FAQ 
Создай 5 вопросов FAQ для интернет-магазина электроники.
Верни результат в формате JSON:

{
  "faq": [
    {
      "id": 1,
      "question": "вопрос",
      "answer": "ответ (2-3 предложения)",
      "category": "доставка" | "оплата" | "гарантия" | "возврат"
    }
  ]
}

Требования:
- Минимум по одному вопросу из каждой категории
- Ответы должны быть конкретными и полезными
- Язык: русский

Верни ТОЛЬКО JSON.
Массовая обработка данных
Классификация списка товаров 
Классифицируй товары по категориям и верни JSON:

{
  "items": [
    {
      "name": "название товара",
      "category": "категория",
      "subcategory": "подкатегория",
      "price_segment": "budget" | "mid" | "premium"
    }
  ],
  "statistics": {
    "total": число товаров,
    "by_category": {"категория": количество}
  }
}

Товары:
1. iPhone 15 Pro - 129990 руб
2. Xiaomi Redmi Note 12 - 15990 руб
3. AirPods Pro 2 - 24990 руб
4. Samsung Galaxy Tab S9 - 89990 руб
5. Logitech MX Master 3 - 8990 руб

Верни ТОЛЬКО JSON.

Лучшие практики

Советы по работе с JSON
1
Всегда указывайте "Верни ТОЛЬКО JSON"

Без этого AI может добавить пояснения, которые сломают парсинг

2
Указывайте типы данных в схеме

"price": число (number), "name": строка (string)

3
Определяйте допустимые значения для enum

"status": "active" | "inactive" | "pending"

4
Указывайте, что делать с отсутствующими данными

"email": "email или null, если не найден"

5
Приводите пример для сложных структур

AI лучше следует конкретному примеру, чем абстрактной схеме

Частые ошибки
  • Не указан формат вывода — AI добавляет текстовые пояснения
  • Слишком сложная схема — AI путается в вложенности
  • Не указаны типы данных — числа приходят как строки
  • Нет обработки null — AI выдумывает данные вместо null
