Форматирование вывода

Markdown, таблицы, списки — контроль формата ответов AI

Цели урока
  • Освоить Markdown-форматирование
  • Научиться запрашивать таблицы
  • Контролировать длину и структуру ответа

Markdown в промптах

AI отлично понимает Markdown и может форматировать ответы с его помощью:

Запрос Markdown-форматирования 
Напиши обзор продукта в формате Markdown:

Используй:
- # для заголовка
- ## для подзаголовков
- **жирный** для ключевых терминов
- - для списков
- > для цитат или важных замечаний
- `код` для технических терминов

Продукт: MacBook Pro M3

Форматы вывода

Таблицы
Запрос таблицы 
Сравни 3 ноутбука и представь результат в виде
Markdown-таблицы с колонками:

| Модель | Цена | Процессор | RAM | Оценка |

Ноутбуки: MacBook Air M3, Dell XPS 15, ThinkPad X1 Carbon
Нумерованные списки
Пошаговая инструкция 
Объясни, как настроить Git, в виде нумерованного списка.

Формат каждого шага:
1. **Название шага**
   Краткое описание (1-2 предложения)
   ```команда если есть```

Количество шагов: 5-7
Структурированные секции
Отчёт с секциями 
Напиши бизнес-отчёт со следующей структурой:

## Резюме
(2-3 предложения)

## Ключевые метрики
(таблица или список)

## Анализ
(3-4 абзаца)

## Рекомендации
(нумерованный список из 3-5 пунктов)

## Следующие шаги
(чеклист с [ ])

Контроль длины

Инструкция Результат
"В одном предложении" ~15-25 слов
"Кратко, 2-3 предложения" ~50-75 слов
"Не более 100 слов" Точный контроль
"Развёрнутый ответ, 300-400 слов" Полноценный текст
"Список из 5 пунктов" Контроль количества

Специальные форматы

Примеры специфических запросов
Pros/Cons: 
Представь в формате:
✅ Плюсы:
- ...
❌ Минусы:
- ...
TLDR + Детали: 
Формат ответа:
**TL;DR:** (1 предложение)

**Подробнее:**
(развёрнутое объяснение)
Q&A формат: 
Ответь в формате FAQ:
**Q:** вопрос
**A:** ответ
