  • Понять, что такое AI агент и чем он отличается от чат-бота

  • Разобрать компоненты и архитектуру агентов

Что такое AI агент?

Определение: AI агент — это автономная система, способная воспринимать окружающую среду, принимать решения и выполнять действия для достижения поставленных целей.
❌ Обычный чат-бот
  • Отвечает на вопросы
  • Не имеет инструментов
  • Не запоминает контекст
  • Не принимает решения
  • Пассивная роль
✅ AI Агент
  • Решает задачи
  • Использует инструменты (API, БД, код)
  • Хранит долгосрочную память
  • Принимает решения автономно
  • Активная роль

Типы AI агентов

1. Автономные агенты

Работают независимо, без постоянного контроля человека.

✅ Преимущества
  • Высокая эффективность
  • Работа 24/7
  • Быстрое реагирование
  • Масштабируемость
⚠️ Ограничения
  • Сложность настройки
  • Риски при ошибках
  • Требуют мониторинга
  • Ограниченная гибкость

Примеры: Торговые боты, системы мониторинга, автоматические отчеты

2. Управляемые агенты

Работают под контролем человека, выполняют задачи по запросу.

✅ Преимущества
  • Простота создания
  • Высокий контроль
  • Безопасность
  • Легкость тестирования
⚠️ Ограничения
  • Требуют участия человека
  • Ограниченная автономность
  • Меньшая эффективность
  • Зависимость от пользователя

Примеры: Custom GPTs, Claude Projects, виртуальные ассистенты

3. Гибридные агенты

Сочетают автономность и управляемость, адаптируются к уровню доверия.

✅ Преимущества
  • Баланс автономности и контроля
  • Высокая гибкость
  • Постепенное обучение
  • Адаптация к потребностям
⚠️ Ограничения
  • Сложность настройки
  • Потенциальная непредсказуемость
  • Требуют тестирования
  • Нужен мониторинг

Примеры: n8n workflows с AI, системы обслуживания клиентов

Tool Use и Function Calling

Tool Use — это способность AI агента использовать внешние инструменты для выполнения задач, которые он не может решить самостоятельно.

Типы инструментов

🔍

Поиск

Поиск в интернете, базах данных, документах

🔢

Вычисления

Математические расчеты, анализ данных

📁

Файлы

Чтение, запись, обработка файлов

🔌

API

Интеграция с внешними сервисами

🔌

MCP

Model Context Protocol — стандарт подключения инструментов к AI агентам

Пример Function Calling

JSON 
{
  "name": "search_weather",
  "description": "Поиск текущей погоды в указанном городе",
  "parameters": {
    "type": "object",
    "properties": {
      "city": {
        "type": "string",
        "description": "Название города"
      },
      "units": {
        "type": "string",
        "enum": ["celsius", "fahrenheit"],
        "description": "Единицы измерения температуры"
      }
    },
    "required": ["city"]
  }
}
💡 Как агент использует инструменты
Пользователь: "Какая погода в Минске?"

Агент (внутренний процесс):
1. Анализирует запрос
2. Определяет, что нужен инструмент search_weather
3. Вызывает функцию: search_weather(city="Минск")
4. Получает результат: {"temp": 5, "condition": "облачно"}
5. Формирует ответ для пользователя

Агент: "В Минске сейчас 5°C, облачно."

Model Context Protocol (MCP)

MCP — открытый стандарт от Anthropic (2024), который стал индустриальным стандартом подключения AI агентов к инструментам, базам данных и сервисам. Поддерживается Claude, ChatGPT, Cursor, n8n и сотнями других платформ.
❌ До MCP
  • Каждая платформа — своя интеграция
  • Сложная настройка API
  • Нет единого стандарта
  • Сложно переключаться между AI
✅ С MCP
  • Один сервер — все AI клиенты
  • Plug-and-play подключение
  • 1000+ готовых MCP серверов
  • GitHub, Notion, Slack, БД — из коробки

Компоненты агента подробнее

🔍 Система восприятия (Perception System)

Функции:

  • Получение данных из внешней среды
  • Обработка различных типов входных данных
  • Фильтрация и предобработка информации

Типы входных данных:

  • Текст: сообщения, документы, команды
  • Изображения: фото, скриншоты, диаграммы
  • Структурированные данные: JSON, CSV, XML
  • События: уведомления, триггеры, API вызовы
🧠 Система обработки (Processing System)

Компоненты:

  • LLM (Large Language Model): основная логика обработки
  • Memory System: хранение контекста и опыта
  • Decision Engine: принятие решений
  • Planning Module: планирование действий

Функции:

  • Анализ полученных данных
  • Принятие решений на основе контекста
  • Планирование последовательности действий
  • Обучение и адаптация
⚡ Система действий (Action System)

Типы действий:

  • Коммуникация: отправка сообщений, уведомлений
  • Интеграции: вызовы API, работа с базами данных
  • Генерация контента: создание текстов, изображений, кода
  • Автоматизация: запуск процессов, управление системами

Функции:

  • Выполнение решений системы обработки
  • Взаимодействие с внешними системами
  • Мониторинг выполнения действий

  • Git интеграция

  • Заполнение форм, бронирование

  • Выполнение задач в интерфейсе
🛠 Инструменты
  • Браузер (Computer Use)

Проверка знаний

Что является ключевым отличием AI агента от обычного чат-бота?
Способность понимать естественный язык
Способность использовать инструменты и принимать решения автономно
Более быстрая генерация ответов
Возможность работать в веб-браузере
📝 Ключевые выводы
  • AI агент — это система, способная воспринимать среду, принимать решения и выполнять действия
  • Три типа агентов: автономные, управляемые и гибридные
  • Компоненты: восприятие, обработка (LLM + память), действие
  • Tool Use позволяет агентам выходить за рамки текстовой генерации
  • Function Calling — механизм вызова внешних функций
  • MCP — единый стандарт подключения инструментов к любому AI агенту (2024+)
  • Тренд 2026: multi-agent системы и Computer Use — агенты управляют браузером и приложениями
