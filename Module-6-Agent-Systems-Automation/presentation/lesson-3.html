Что такое n8n?

n8n — open-source платформа для автоматизации workflows. Позволяет соединять различные сервисы и API без программирования через визуальный drag-and-drop интерфейс. В версии 1.x появились нативные AI Agent ноды — создавайте полноценных агентов с памятью и инструментами прямо в n8n, без кода.

Типы нодов

Trigger Nodes (Триггеры)

Запускают выполнение workflow:

  • Manual Trigger — запуск вручную для тестирования
  • Webhook — HTTP запросы как триггер (REST API)
  • Schedule Trigger — запуск по расписанию (cron-синтаксис)
  • Email Trigger — обработка входящих писем
Processing Nodes (Обработчики)

Обрабатывают и трансформируют данные:

  • Function Node — выполнение JavaScript кода
  • Code Node — выполнение Python кода
  • Set Node — установка значений полей
  • IF Node — условная логика и разветвление
AI/ML Nodes

Интеграция с AI сервисами:

  • AI Agent Node — нативный агент с памятью, инструментами и LLM (новое в v1.x)
  • OpenAI — GPT-5.2, o3 для текста и анализа
  • Anthropic Claude — Claude Sonnet/Opus 4.6, нативная нода
  • Google Gemini — Gemini 2.5 Pro/Flash, нативная нода
  • MCP Client — подключение MCP серверов к агентам
  • HTTP Request — универсальный API для любых AI сервисов
Action Nodes (Действия)

Выполняют конечные действия:

  • HTTP Request — отправка HTTP запросов
  • Email — отправка писем
  • Slack — сообщения в каналы
  • Google Sheets — работа с таблицами

Установка n8n

Self-hosted (npm)
# Глобальная установка
npm install n8n -g

# Запуск
n8n start

# С аутентификацией
N8N_BASIC_AUTH_ACTIVE=true \
N8N_BASIC_AUTH_USER=admin \
N8N_BASIC_AUTH_PASSWORD=password \
n8n start
Docker Compose
version: '3.8'
services:
  n8n:
    image: n8nio/n8n
    ports:
      - "5678:5678"
    environment:
      - N8N_BASIC_AUTH_ACTIVE=true
      - N8N_BASIC_AUTH_USER=admin
    volumes:
      - n8n_data:/home/node/.n8n
n8n Cloud — облачная версия. В 2025-2026 получила нативные AI Agent ноды и поддержку MCP. Регистрация: n8n.io

Пример 3: Классификация заявок

Support

Автоматическая маршрутизация

Структура:

Webhook → Function (Classification) → IF → Email/Slack

Function Node (классификация):

JavaScript 
// Анализ заявки и определение приоритета
const request = $input.first().json;
const keywords = request.message.toLowerCase();

let priority = 'low';
let department = 'general';

// Определение приоритета
if (keywords.includes('срочно') || keywords.includes('критично')) {
  priority = 'high';
} else if (keywords.includes('важно') || keywords.includes('проблема')) {
  priority = 'medium';
}

// Определение отдела
if (keywords.includes('техническая') || keywords.includes('баг')) {
  department = 'technical';
} else if (keywords.includes('продажи') || keywords.includes('заказ')) {
  department = 'sales';
} else if (keywords.includes('поддержка') || keywords.includes('помощь')) {
  department = 'support';
}

return {
  json: {
    ...request,
    priority: priority,
    department: department,
    processed_at: new Date().toISOString()
  }
};
